API Update Breaking Change

OpenAI Agents SDK v0.16.0: デフォルトモデル変更と新機能追加

2026-05-07

概要

!

デフォルトモデルが gpt-5.4-mini に変更 (破壊的変更)

SDK のデフォルトモデルが gpt-4.1 から gpt-5.4-mini に変更。GPT-5 固有のデフォルト値 (reasoning.effort="none", verbosity="low") が暗黙的に適用されるため、既存エージェントの動作に影響する可能性あり。

∞

max_turns=None でターン制限を無効化

長時間のマルチステップタスクに対応するため、ラン実行のターン数制限を無効化するオプションが追加。デフォルト (10) は維持。

⚙

ToolExecutionConfig で並行性を制御

SDK 側でのローカル関数ツール実行の並行数を max_function_tool_concurrency で制限可能に。プロバイダー側の parallel_tool_calls とは独立。

🔗

MCP ツール名にサーバー名を付与

include_server_in_tool_names オプションにより、複数 MCP サーバー間のツール名衝突を防止可能に。

Before / After

Before (v0.15.x)

  • デフォルトモデル: gpt-4.1
  • max_turns: 常に数値指定が必要
  • ツール並行実行: 制御不可
  • MCP ツール名: サーバー名なし

After (v0.16.0)

  • デフォルトモデル: gpt-5.4-mini
  • max_turns=None で無制限実行可
  • ToolExecutionConfig で並行数制御
  • サーバー名プレフィックスで衝突防止

新機能

デフォルトモデル変更

gpt-4.1 から gpt-5.4-mini へ。GPT-5 系のデフォルト設定が自動適用される。明示的にモデル指定または環境変数 OPENAI_DEFAULT_MODEL で上書き可能。

ターン制限の無効化

max_turns=None を指定することで DEFAULT_MAX_TURNS (10) の制限を撤廃。複雑なリサーチやマルチステップワークフローに対応。

ツール実行スケジューラ

ToolExecutionConfig の max_function_tool_concurrency で SDK 側のツール実行並行数を制御。レート制限のある外部 API 呼び出しに有効。

MCP サーバー名付与

MCPConfig の include_server_in_tool_names=True でツール名に MCP サーバー名を含め、複数サーバー間の名前衝突を防止。

コードサンプル

モデルの明示的指定 (破壊的変更への対応)

from agents import Agent # gpt-4.1 を明示的に指定して従来動作を維持 agent = Agent(name="Assistant", model="gpt-4.1")

ツール実行の並行性制御

from agents import Agent, Runner from agents.run import RunConfig, ToolExecutionConfig run_config = RunConfig( tool_execution_config=ToolExecutionConfig( max_function_tool_concurrency=3 ) ) result = await Runner.run( agent, input="task", run_config=run_config )

max_turns の無効化

# ターン制限なしで実行 result = await Runner.run( agent, input="complex research", max_turns=None )

ユースケース

長時間リサーチエージェント

max_turns=None により、Web 検索とドキュメント分析を繰り返す複雑なリサーチタスクをターン制限なしで実行可能。

レート制限のある外部 API 連携

ToolExecutionConfig で並行数を制限し、外部 API のレート制限を超えないよう SDK レベルで制御。

マルチサーバー MCP 環境

include_server_in_tool_names でサーバー名をプレフィックスとして付与し、ファイルシステムサーバーと DB サーバーの "read" ツールの衝突を防止。

バグ修正

🔧

Chat completions ツール呼び出し出力インデックスの安定化

ツール呼び出し結果の出力インデックスが安定するよう修正。

🔒

外部シンボリックリンクターゲットの拒否

hydrate 処理時に外部を指すシンボリックリンクターゲットを拒否するよう修正 (セキュリティ強化)。

🛠

Permissions のハッシュ可能化

Permissions クラスを User や Group と同様にハッシュ可能に修正。
