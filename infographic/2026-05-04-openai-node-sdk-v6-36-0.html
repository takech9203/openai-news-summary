API UPDATE

OpenAI Node SDK v6.36.0: Admin API Keys のエンドポイント単位スコープ

2026-05-04

概要

🔑

Admin API Keys のエンドポイント単位スコープ (per endpoint)

特定のエンドポイントのみにアクセスを許可する Admin API Key を作成可能に。最小権限の原則に基づくセキュリティ強化。

🏷

group_type / user メタデータフィールドの追加

Admin リソースに種類分類とユーザー情報を関連付け。大規模組織でのリソース管理効率化と監査の改善。

🔐

認証ヘッダー選択の厳格化

Admin API と標準 API の混在使用時に適切な認証ヘッダーを自動選択。認証エラーの解消。

🤝

Python SDK v2.34.0 との協調リリース

Node SDK と Python SDK が同日リリース。多言語プロジェクトで一貫した機能セットを提供。

🛡

最小権限の原則に基づくキー管理

キー漏洩時の影響範囲を最小化し、マイクロサービスごとに必要最小限の権限のみを付与可能。

主な機能

🔑 Scoped Admin Keys

特定エンドポイントのみにアクセス許可するキーを作成。キー漏洩時の影響範囲を最小化。

🏷 group_type フィールド

Admin リソースの種類分類。大規模組織でのリソース管理効率化。

👤 user メタデータ

Admin リソースにユーザー情報を関連付け。追跡と監査の改善。

🔐 認証厳格化

適切な認証ヘッダーの自動選択。Admin API と標準 API の混在使用時のエラー解消。

🤝 Python SDK パリティ

v2.34.0 と同日リリース。多言語プロジェクトで一貫した体験。

コードサンプル

TypeScript Scoped Admin API Key Creation 
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.OPENAI_ADMIN_KEY,
});

// v6.36.0: エンドポイント単位スコープ
const scopedKey = await client.admin.apiKeys.create({
  name: "chat-only-key",
  scoped_endpoints: [
    "/v1/chat/completions",
    "/v1/models",
  ],
});
console.log(`Scoped Key: ${scopedKey.key}`);
TypeScript Permission Denied Handling 
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: "sk-scoped-xxxxx",
});

try {
  await client.images.generate({
    model: "dall-e-3",
    prompt: "A sunset",
  });
} catch (error) {
  if (error instanceof OpenAI.PermissionDeniedError) {
    console.error("Access denied for this endpoint");
  }
}

ユースケース

🏗 マイクロサービス

各サービスに必要最小限の権限のみ付与。Chat サービスには /v1/chat/completions のみ、画像生成サービスには /v1/images のみといった細粒度のアクセス制御が可能。

🔍 セキュリティ監査

キー漏洩時の影響範囲限定。スコープ付きキーにより、万が一キーが流出しても特定エンドポイント以外へのアクセスは不可能。

🏢 大規模組織管理

group_type で効率的なリソース分類。組織内の部門やプロジェクトごとにリソースをグループ化し、管理を効率化。
