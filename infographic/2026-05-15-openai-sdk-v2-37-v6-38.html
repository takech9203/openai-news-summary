OpenAI Python SDK v2.37.0 / Node SDK v6.38.0
Responses Compact メソッドに service_tier パラメータを追加

2026-05-15

📋 概要

1

Responses Compact メソッドに service_tier パラメータを追加 (Python / Node 両 SDK 共通)

2

service_tier の選択肢: "auto" (自動選択)、"default" (標準)、"flex" (低コスト) でコストとレイテンシを制御

3

compact メソッドで会話履歴を圧縮し、トークン消費を削減。長い会話のコスト最適化に貢献

4

Python SDK: Pydantic イテレータの即時バリデーション、ワークロード ID プロバイダ認証の簡素化、f-string バグ修正

🔄 Before / After 比較

BEFORE

compact メソッドはティア指定不可

常にデフォルトティアで処理されるため、コストやレイテンシの制御ができなかった。全ての圧縮リクエストが同一条件で処理。

AFTER

service_tier で柔軟に制御可能

"auto" / "default" / "flex" を指定することで、ユースケースに応じたコストとレイテンシのバランスを選択可能に。

⚙ service_tier オプション

🤖

auto

"auto"

システムが最適なティアを自動選択。負荷状況やリクエストの優先度に基づき動的にルーティング。

⭐

default

"default"

標準ティアで処理。安定したレイテンシと予測可能なコストを提供。本番環境の一般的な選択肢。

💰

flex

"flex"

低コストティア。レイテンシを許容する代わりにコストを削減。バッチ処理や非リアルタイム用途に最適。

💻 コードサンプル

Python Responses Compact with service_tier 
from openai import OpenAI

client = OpenAI()

# 会話履歴を圧縮 (service_tier を指定)
compact_response = client.responses.compact(
    response_id=response.id,
    service_tier="auto",  # "auto" | "default" | "flex"
)

print(f"圧縮後のレスポンス ID: {compact_response.id}")
print(f"使用されたサービスティア: {compact_response.service_tier}")
TypeScript Responses Compact with service_tier 
import OpenAI from "openai";

const client = new OpenAI();

const compactResponse = await client.responses.compact({
  response_id: response.id,
  service_tier: "auto",
});

console.log(`Service tier used: ${compactResponse.service_tier}`);

📝 Python SDK v2.37.0 変更一覧

Feature Pydantic イテレータの即時バリデーション対応

🚀 開発者への影響

  • コスト最適化: flex ティアを活用することで、バッチ処理やバックグラウンドジョブの圧縮コストを削減可能
  • レイテンシ制御: リアルタイム会話では default、非同期処理では flex と使い分けることでパフォーマンスを最適化
  • 認証の簡素化: Python SDK でワークロード ID プロバイダ認証から client_id が不要に。設定コードが簡潔化
  • 型安全性の向上: Pydantic イテレータの即時バリデーションにより、不正なデータを早期に検出可能
  • 移行コスト: 既存コードはそのまま動作 (service_tier はオプション)。段階的に導入可能
