📦

コンテナベースの完全隔離

ゼロトラストネットワーク

エージェント行動の完全可視化

セキュリティの 4 つの柱

📦 サンドボックス

  • 読み取り専用ルート FS

コード変更に対する多段階レビュープロセス

  • 多段階承認モデル
  • 変更影響分析
  • PR ベースレビュー
  • ロールバック機能
  • 監査証跡

🌐 ネットワークポリシー

ゼロトラスト原則に基づくネットワーク制御

  • デフォルト拒否
  • ホワイトリスト方式
  • DNS フィルタリング
  • 内部ネットワーク分離
  • 送信データ検査

📡 エージェントネイティブテレメトリ

AI エージェント専用の可観測性基盤

  • アクションレベルトレーシング
  • 推論キャプチャ

  • リソースモニタリング
  • クロスエージェント相関

コードサンプル

YAML codex-sandbox-policy.yaml - サンドボックスポリシー定義 
# codex-sandbox-policy.yaml
apiVersion: codex.openai.com/v1
kind: SandboxPolicy
metadata:
  name: production-codex-agent
spec:
  container:
    image: codex-runtime:latest
    readOnlyRootFilesystem: true
    resources:
      limits:
        cpu: "4"
        memory: "8Gi"
        ephemeral-storage: "20Gi"
  security:
    runAsNonRoot: true
    allowPrivilegeEscalation: false
    capabilities:
      drop: ["ALL"]
  network:
    defaultPolicy: deny
    allowedEndpoints:
      - host: "pypi.org"
        port: 443
        protocol: HTTPS
Python 承認ポリシー付きタスク作成 API 
from openai import OpenAI

client = OpenAI()

task = client.codex.tasks.create(
    description="認証モジュールのリファクタリング",
    repository="org/backend-service",
    approval_policy={
        "level": "multi_reviewer",
        "required_approvers": 2,
        "auto_approve_paths": ["tests/*", "docs/*"],
        "block_paths": ["*.env", "secrets/*"],
    },
    sandbox_policy="production-codex-agent",
)

ユースケース

エンタープライズコード生成

本番環境のコードベースに対して Codex エージェントがリファクタリングや機能追加を実行する際、サンドボックス隔離と承認ワークフローにより、意図しない変更やセキュリティリスクを防止します。

依存関係の安全な管理

ネットワークポリシーのホワイトリスト制御により、Codex が外部パッケージをインストールする際に許可済みレジストリ (PyPI, npm, GitHub) のみにアクセスを限定し、サプライチェーン攻撃を防止します。

セキュリティインシデント対応

エージェントネイティブテレメトリにより、Codex の全アクションがトレース・記録されるため、異常な動作を即座に検知し、SIEM / SOC との連携で迅速なインシデント対応が可能です。
