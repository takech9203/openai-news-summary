ChatGPT Lockdown Mode と Elevated Risk Labels: プロンプトインジェクション対策の新セキュリティレイヤー

2026-06-06 OpenAI News (Safety) Lockdown Mode

概要

OpenAI は 2026 年 6 月 6 日、ChatGPT に「Lockdown Mode」と「Elevated Risk Labels」という 2 つの新しいセキュリティ機能を導入した。Lockdown Mode は、機密データを扱うユーザーや組織を対象に、プロンプトインジェクション攻撃によるデータ流出リスクを軽減するための厳格な保護機能である。

外部コンテンツとの接触を最小限に抑えることで攻撃面を縮小するアプローチを採用し、すべてのユーザー向けではなく、セキュリティを最優先する環境でのオプトイン利用を想定している。ChatGPT Business のセルフサーブアカウントおよび対象個人アカウントへ順次展開される。

キーポイント

🛡
攻撃面の縮小: Web ブラウジング、画像取得、Deep Research、Agent モードを無効化し、外部からの悪意ある指示の注入経路を物理的に遮断
🔒
データ流出防止: 機密データを扱う組織向けに設計された、プロンプトインジェクションによる情報漏洩リスクの大幅低減
⚠
Elevated Risk Labels: コンテンツ内の潜在的なセキュリティリスクを検出し、ユーザーに警告を表示するコンパニオン機能
🏢
対象限定のオプトイン: 全ユーザー一律ではなく、ChatGPT Business セルフサーブアカウントと対象個人アカウントに段階的展開
⚡
透明な限界開示: キャッシュコンテンツやアップロードファイル経由の攻撃は依然として残存リスクであることを OpenAI が明確に認識

Lockdown Mode で無効化される機能

🌐

Web ブラウジング

ライブ Web ブラウジングを完全に無効化。キャッシュされたコンテンツのみアクセス可能。

無効化
🖼

Web 画像取得

Web からの画像取得・表示を遮断。ただし DALL-E による画像生成は引き続き利用可能。

無効化
🔍

Deep Research

広範な Web リサーチ機能を無効化し、外部コンテンツからの攻撃経路を遮断。

無効化
🤖

Agent モード

自律的にタスクを実行する Agent モードを無効化し、意図しない外部アクセスを防止。

無効化

Before / After 比較

Lockdown Mode なし (従来)

  • Web ページ内の隠し指示がモデルに到達
  • 画像内に埋め込まれた悪意あるテキストの読み取り
  • Deep Research 中の汚染コンテンツ取得
  • Agent が悪意あるサイトにアクセス
  • データ流出のリスクが広範に存在

Lockdown Mode 有効

  • 外部 Web アクセス経路を物理的に遮断
  • Web 画像経由の攻撃を根本的にブロック
  • 深層リサーチによるリスク排除
  • Agent の意図しない外部アクセスを防止
  • 攻撃面を大幅に縮小

残存リスク (OpenAI が開示)

キャッシュされた Web コンテンツ: ライブブラウジングは無効でも、事前にキャッシュされたコンテンツに悪意ある指示が含まれている可能性がある
アップロードされたファイル: ユーザーがアップロードするドキュメントや画像に悪意あるプロンプトが埋め込まれている場合は防御対象外
会話内の注入: ユーザーが意図せずコピー&ペーストした悪意あるテキストなど、直接入力経由の攻撃は残存
重要: Lockdown Mode は多層防御の一要素 OpenAI は、本機能がすべてのプロンプトインジェクションリスクを排除するものではないことを明確に認めている。完全な防御を保証するものではなく、攻撃面の大幅な縮小を目的とした追加的なセキュリティレイヤーとして位置づけられる。

想定ユースケース

🏦 金融機関の機密データ分析

顧客の財務情報や取引データを ChatGPT で分析する際、外部への情報漏洩を防止

  • Web ブラウジング経由での情報流出リスクを排除
  • Agent モードによる意図しない外部通信を防止
  • 規制遵守 (GDPR、金融規制) のための追加保護レイヤー

⚖ 法律事務所の契約書レビュー

クライアントの機密契約書を ChatGPT でレビューする際のセキュリティ強化

  • 弁護士 - クライアント秘匿特権の保護
  • 契約書内容の外部漏洩リスクを最小化
  • アップロードファイルへの Elevated Risk Labels による追加警告

🏥 医療機関の患者データ処理

HIPAA 対象の患者データを含む分析業務でのセキュリティ確保

  • 患者の個人健康情報 (PHI) の外部流出防止
  • 医療記録の分析時における厳格なデータ保護
  • コンプライアンス要件への適合を支援

開発者・組織への影響

🔧

API 設計への示唆

外部アクセス制限をセキュリティ対策として公式に位置づけ。API レベルでの同様の機能提供が今後期待される。

📋

セキュリティポリシー策定

ChatGPT Business 管理者は、組織全体 vs ユーザー単位での Lockdown Mode 適用方針の決定が必要。

⚖

コンプライアンス対応

規制対象データを扱う業務での追加保護として、監査・レポーティングに活用可能。
