スパース回路によるニューラルネットワークの理解

2026-07-21 | OpenAI Research - Mechanistic Interpretability

概要

🔬

スパース回路の発見

大規模言語モデル内部で特定の計算や振る舞いを担う最小限のコンポーネント群 (スパース回路) を特定・分析する新手法を提案。メカニスティック・インタープリタビリティの重要な進展を示す。

🧠

ネットワークの「配線図」を読み解く

モデル全体の重みを解析するのではなく、特定の振る舞いに関与する少数のニューロン、アテンションヘッド、MLP 層のサブセットを特定し、内部構造を理解可能にする。

🛡️

AI 安全性研究の中核

OpenAI が推進する AI 安全性研究プログラムの中核に位置し、モデルの内部状態を監視・理解するための技術的基盤を提供する。

📊

極めてスパースな回路構造

個々のタスクや振る舞いに実際に関与するのは全パラメータのごく一部 (0.1% ~ 10%) であり、ネットワークは高度にモジュール化された構造を持つ。

🔗

スケーラブルな手法

小規模モデルから GPT-5.x シリーズまで適用可能な階層的探索手法を採用し、大規模モデルでも効率的に回路を発見する。

スパース回路とは

🎯 最小性

特定の振る舞いを再現するために必要最小限のコンポーネントで構成される。例: 間接オブジェクト識別回路 -「Mary told John that ___」の空欄に she を生成する最小構成。

🔀 機能的独立性

回路は他のネットワーク部分とある程度独立して機能し、切り離しても対象の計算が保持される。例: 事実想起回路 -「東京は日本の首都」を検索・出力する独立した回路。

🧩 構成性

複数のスパース回路が組み合わさって、より複雑な振る舞いを実現する。例: 算術演算回路 - 数値の加減算を行う際に活性化するニューロン群の組み合わせ。

👁️ 解釈可能性

回路の各コンポーネントに人間が理解可能な意味的役割を割り当てられる。例: 構文解析回路 - 文法構造を追跡し、正しい閉じ括弧や動詞活用を生成する回路。

研究手法

⚡ 活性化パッチング

ネットワーク内の特定コンポーネントの活性化値を別の入力に対する値で置き換え、出力への影響を測定。影響が大きいコンポーネントが回路の一部であると特定する。

🔍 回路発見アルゴリズム

エッジレベル探索、属性スコアリング (勾配ベース)、L1 正則化やマスク学習によるスパース最適化を組み合わせて最小限の回路を発見する。

🏗️ スパースオートエンコーダ

OpenAI の先行研究である SAE を用いてモデルの中間表現を解釈可能な特徴量に分解し、回路の構成要素をより高いレベルの概念として理解する。

📈 スケーラブルな発見手法

階層的な探索手法を採用: まず層レベルで重要な範囲を絞り込み、次にアテンションヘッドや MLP ニューロンレベルで精密な回路を特定する段階的アプローチ。

回路発見パイプライン - コード例

Python discover_circuit - スパース回路発見関数 
def discover_circuit(
    model: torch.nn.Module,
    target_behavior: callable,
    dataset: list[dict],
    sparsity_target: float = 0.01,  # 全パラメータの 1% 以下
) -> SparseCircuit:
    """
    対象の振る舞いを担うスパース回路を発見する。

    Args:
        model: 解析対象の Transformer モデル
        target_behavior: 対象の振る舞いを定義する関数
        dataset: 振る舞いを観察するためのデータセット
        sparsity_target: 目標スパース度 (低いほど少ないコンポーネント)

    Returns:
        SparseCircuit: 発見された回路
    """
    # Phase 1: 粗い探索 - 層レベルの重要度推定
    layer_importance = estimate_layer_importance(
        model, target_behavior, dataset
    )

    # Phase 2: コンポーネントレベルの活性化パッチング
    component_scores = activation_patching(
        model, target_behavior, dataset,
        target_layers=get_important_layers(layer_importance)
    )

    # Phase 3: スパース最適化による回路の精緻化
    circuit_mask = optimize_sparse_mask(
        model, target_behavior, dataset,
        initial_scores=component_scores,
        sparsity_target=sparsity_target
    )

    # Phase 4: 回路の検証と忠実性評価
    faithfulness = evaluate_circuit_faithfulness(
        model, circuit_mask, target_behavior, dataset
    )

    return SparseCircuit(
        name=f"circuit_{target_behavior.__name__}",
        components=mask_to_components(circuit_mask, component_scores),
        faithfulness_score=faithfulness,
        sparsity=circuit_mask.sum() / circuit_mask.numel()
    )

主要な発見

🧱 回路のモジュール性

大規模言語モデル内部の計算は高度にモジュール化されたスパース回路によって組織されている。ネットワークが「スープ状」に情報を分散処理しているのではなく、特定の機能単位が存在することを示唆する。

🌐 回路の普遍性

異なるスケールのモデル (数十億から数百億パラメータ) において、類似の機能を担う回路が類似の構造を持つ。ネットワークアーキテクチャに依存しない「計算の普遍的パターン」の存在を示唆する。

🔄 スーパーポジションとの関係

一つのニューロンが複数の特徴を同時にエンコードする「スーパーポジション」は、スパース回路の視点から見ると、複数の回路が同一のコンポーネントを共有する形で実現されている。

🛡️ 安全性回路の特定

モデルが有害な出力を拒否する際に活性化する「安全性回路」や、指示に従う際に活性化する「指示追従回路」が特定された。AI 安全性のモニタリングに直接応用可能な知見。

開発者への影響

🛡️ AI 安全性への直接的貢献

特定の入力に対してどの回路が活性化するかを理解することで、モデルが予期しない出力を生成する原因を事前に特定し防止策を講じることが可能になる。安全性回路を特定・分析し、バイパスされにくい設計への改善や、抽象的なアライメント概念を回路レベルの検証に落とし込むことが可能になる。

🔧 モデルのデバッグと改善

モデルが誤った回答を生成する際に誤作動している回路を特定してターゲットを絞った修正が可能になる。バイアスがどの回路に起因するかを特定し、そのコンポーネントを修正・再訓練することで、よりフェアなモデルを構築できる。対象タスクに関連する回路のみを効率的にファインチューニングする手法への応用も期待される。

🔮 今後の研究方向性

推論時にどの回路が活性化しているかをリアルタイムで監視するシステムの構築、特定の回路を選択的に活性化・抑制してモデルの振る舞いを精密に制御する技術、安全性上の問題を引き起こす回路を自動的に検出・修正するメカニズムの開発が見込まれる。

🏛️ 安全性研究スタック内の位置づけ

CoT モニタリングや Confessions メカニズムと相補的に機能し、モデルの内部状態を多角的に理解・監視するための技術的基盤を構成する。モデルの能力が急速に向上する現在、その内部動作を理解し制御する能力を維持することが安全な AI 開発の根幹である。
