Before / After

Before (v0.134.0)

  • 基本的な環境チェックのみ
  • リモート接続状態が不透明
  • Vim 基本移動コマンドのみ
  • ファイルベースのメモリ管理
  • Python SDK にプリセットなし
  • レガシー設定パスが残存

After (v0.135.0)

  • 5 カテゴリの包括的診断
  • /status でリモート詳細表示
  • フルテキストオブジェクト対応
  • 専用 SQLite DB に移行
  • Sandbox プリセット API 提供
  • レガシーパス完全削除

主な新機能

🏥 codex doctor

環境、Git、ターミナル、App-Server、スレッドインベントリの 5 カテゴリで包括的な診断情報を収集。サポートケースの迅速な解決を支援する。

🌐 /status リモート情報

リモートトランスポート経由の接続時に、サーバーバージョン、ホスト名、プロトコル、レイテンシを表示。

⌨️ Vim テキストオブジェクト

iw、aw、i"、a(、i{、a{ 等の完全なテキストオブジェクト編集。word-end/line-end 動作改善と設定可能な中断バインディング。

🔐 名前付きプロファイル

/permissions が名前付きパーミッションプロファイルを理解し、設定済みカスタムプロファイルを一覧表示。

🐚 zsh ヘルパー自動検出

パッケージビルドがバンドル済みパッチ zsh ヘルパーを macOS/Linux で自動検出。システム zsh に依存しない動作を実現。

🐍 Sandbox プリセット

Python SDK で Thread API と Turn API の両方から利用可能な Sandbox プリセット (STANDARD / RESTRICTED / PERMISSIVE)。

バグ修正

  • Markdown テーブルとマルチラインリストの TUI レンダリング改善 (カラムサイズの最適化、アプリスタイルのテーブル表示)
  • macOS と Zellij での TUI 出力安定化 (stderr/composer 破損と raw-output オーバーラップの解消)
  • スラッシュコマンド補完がインライン引数を受け付けるコマンドの既存ドラフトテキストを保持するように修正
  • 古い tmux/iTerm コントロールモードセッションでの Ctrl-C ハンドリング回復
  • アクセス不能または無効化されたアプリをメンション候補から除外
  • Resume フローが非対話的 exec セッションを含み、アイドルキャッシュスレッドの cwd オーバーライドを尊重するように修正

Python Python SDK Sandbox プリセット 
from codex_sdk import CodexClient, Sandbox

# クライアントの初期化
client = CodexClient()

# Sandbox プリセットを使用してスレッドを作成
thread = client.threads.create(
    sandbox=Sandbox.STANDARD,  # 標準サンドボックス
)

# ターン API でもプリセットを使用可能
response = client.threads.turns.create(
    thread_id=thread.id,
    message="Refactor the authentication module",
    sandbox=Sandbox.RESTRICTED,  # 制限付き
)

# 利用可能なプリセット:
# Sandbox.STANDARD    - 標準的な開発用
# Sandbox.RESTRICTED  - ネットワークアクセスなし
# Sandbox.PERMISSIVE  - 高信頼環境向け

開発者への影響

安定版としての本番利用が可能に

アルファ版で試験的に提供されていた全機能が安定版品質で利用可能。セマンティックバージョニングに準拠し、CI/CD パイプラインへの組み込みが安全に行える。

環境診断によるトラブルシューティングの迅速化

codex doctor の強化された 5 カテゴリ診断レポートにより、問題報告時に必要な情報が自動収集される。サポートチームとのコミュニケーションコストが削減される。

Vim ユーザーの生産性向上

テキストオブジェクトバインディングにより、Vim ユーザーが筋肉記憶を活かしたまま TUI を操作可能。カスタマイズ可能な中断キーバインドも提供。

Python 開発者向け SDK の充実

Sandbox プリセット API により、セキュリティレベルを明示的に指定してスレッドを作成可能。ドキュメントとサンプルノートブックも更新済み。

メモリシステムの信頼性向上

SQLite への移行により、メモリデータの整合性とパフォーマンスが改善。既存データは自動マイグレーションされるため手動作業不要。
