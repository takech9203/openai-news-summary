OpenAI Agents SDK v0.16.1: 安定性・セキュリティ・データ整合性に関するバグ修正リリース

2026-05-07

v0.16.0 に対する 7 件のバグ修正。ストリーミング出力の安定性、MCP セキュリティポリシー検証、セッション管理の堅牢性を向上。

3

破壊的変更

ストリーミング出力インデックスの安定化

Chat Completions API のストリーミングレスポンスにおいて、ツール呼び出し結果のインデックスが変動する問題を修正。エージェントの応答組み立て信頼性が向上。

🔒

MCP セキュリティポリシーの適切な検証

require_approval ポリシー設定が正しく検証されるようになり、意図しないツール実行の自動承認を防止。MCP ツールのセキュリティガバナンスを強化。

💾

セッション管理の堅牢性向上

コンパクション失敗後の履歴復元、Dapr セッション状態の破損検出、暗号化アイテムの正確なカウントなど、分散環境での長期運用における信頼性を大幅改善。

安定性の修正

Stability

ストリーム出力インデックス安定化

ストリーミング中にツール呼び出し結果のインデックスが変動し、エージェントの応答組み立てに失敗するケースを修正。

#3109

セッション履歴のコンパクション復元

コンパクション処理の置換操作が失敗した場合にセッション履歴が消失する問題を修正。失敗時に元の履歴が正しく復元される。

#3116

セキュリティの修正

Security

MCP require_approval ポリシー検証

MCP ツール実行時の require_approval ポリシー設定が正しく検証されるようになった。不正なポリシー設定の黙認を防止。

#3168

非オブジェクト型 Function Tool 入力の拒否

Function Tool に渡される入力 JSON がオブジェクト型でない場合 (配列や文字列が直接渡される場合) に拒否するバリデーションを追加。

データ整合性の修正

Data Integrity

Dapr セッション状態の破損検出

Dapr を使用したセッション状態管理において、破損したセッション状態の更新を拒否。分散環境でのセッション管理の信頼性が向上。

#3171

暗号化セッションアイテムの正確なカウント

セッション制限の計算において、暗号化されたセッションアイテムのうち有効なもののみをカウントするよう修正。利用可能容量の正確性が向上。

#3174

Git 一時クローンの失敗時クリーンアップ

Git リポジトリの一時クローン処理が失敗した際に、不完全な一時ディレクトリが残存する問題を修正。ディスクスペースのリークを防止。

#3170

開発者への影響

ストリーミング利用時

出力インデックスの不安定性により、ツール呼び出しの結果が正しく処理されない可能性がある。ストリーミングモードでエージェントを実行している場合は速やかにアップデートが必要。

MCP ツール利用時

セキュリティポリシーが正しく適用されず、意図しない自動承認が発生する恐れがある。MCP ツールを使用している環境ではセキュリティ上の観点からアップデートが重要。

長時間セッション利用時

コンパクション処理の失敗により、会話コンテキストが消失するリスクがある。長時間のセッションを運用している場合は特に影響が大きい。

Dapr 環境での運用時

破損した状態更新が処理されることで、セッション全体が不正な状態になる恐れがある。分散環境で Agents SDK を使用している場合は最優先でアップデートを推奨。

アップデート方法

本リリースはバグ修正のみであり、API インターフェースや動作仕様の変更は含まれていない。コードの修正は不要。

# pip でのアップデート pip install --upgrade openai-agents # バージョン確認 pip show openai-agents | grep Version # Version: 0.16.1
