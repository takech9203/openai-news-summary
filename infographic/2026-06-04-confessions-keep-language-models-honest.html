Key Points

💬

Confessions メカニズムの導入

言語モデルが自身の不確実性や情報の欠如を積極的に「告白」するメカニズムを導入し、ハルシネーションの影響を軽減する新しいアプローチを提案

🎯

Honesty Reward による強化学習

正直な不確実性表明に対して報酬を与える「Honesty Reward」を RLHF の拡張として導入し、モデルの誠実性を訓練レベルで改善

🔍

Uncertainty Probe による内部状態分析

Transformer の隠れ層活性化パターンから不確実性を検出するプローブを訓練し、モデルの内部的な確信度を定量化

📊

キャリブレーション精度の向上

告白メカニズムを導入したモデルは事実性ベンチマークにおいて信頼性が向上し、ユーザー信頼度も改善されることが示唆

🤝

人間と AI の協働を強化

モデルが「何を知らないか」を明確にすることで、ユーザーは適切な期待値を持って AI と対話でき、意思決定の質が向上

問題設定: 言語モデルの誠実性に関する 3 つの課題

⚠ 過信バイアス

学習データに含まれない情報や曖昧な質問に対しても、あたかも確信があるかのように回答を生成。ユーザーが誤った情報を正しいものとして受け取るリスクが存在する。

⚠ キャリブレーションの欠如

モデルの内部的な確信度と出力の表現が一致しない。確率的に 50% 程度の確信しかない回答でも、断定的な表現で出力されることがある。

⚠ 検出の困難さ

モデルがいつ信頼できる情報を提供し、いつ不確実な情報を生成しているかを外部から判断することが極めて困難である。

告白の 4 つのタイプ

📖 知識の欠如の告白

「この質問に対する正確な情報を持っていません」

学習データに該当する情報が乏しいと推定される場合にトリガーされる

❓ 不確実性の告白

「この回答には確信がなく、複数の解釈が可能です」

モデルの内部表現における不確実性が閾値を超えた場合にトリガーされる

📅 情報源の限界の告白

「この情報は学習データに基づいており、最新の状況を反映していない可能性があります」

事実と意見の区別が曖昧な回答を生成しようとしている場合にトリガーされる

🧩 推論の弱さの告白

「この結論は推測に基づいており、確実な根拠がありません」

論理的推論の連鎖が弱いと内部的に評価された場合にトリガーされる

研究方法論: 3 つのステージ

1

Uncertainty Probing (不確実性信号の学習)

モデルの内部状態から不確実性を検出するためのプロービング手法を適用。隠れ層の活性化パターンから、モデルが確信を持っている場合と持っていない場合の差異を特定するクラシファイアを訓練する。

2

Honesty RL (告白行動の強化学習)

RLHF の拡張として、正直な不確実性表明に対して報酬を与える「Honesty Reward」を導入。モデルが不確実な情報を断定的に述べた場合にペナルティを課し、適切に不確実性を表明した場合に報酬を与える。

3

Calibration Evaluation (キャリブレーション評価)

告白メカニズムを導入したモデルと導入していないモデルを、事実性ベンチマーク (TruthfulQA 等) で比較評価。告白の頻度と実際の正確性の相関を定量的に測定する。

💻 Honesty Reward の概念的な実装 
# Confessions Honesty Reward の概念的な実装
def compute_honesty_reward(
    model_output: str,
    internal_uncertainty: float,
    ground_truth_available: bool,
    ground_truth: str | None = None
) -> float:
    """
    モデルの出力に対する誠実性報酬を計算する。

    Args:
        model_output: モデルの生成テキスト
        internal_uncertainty: モデル内部の不確実性スコア (0-1)
        ground_truth_available: 正解データが利用可能か
        ground_truth: 正解データ (利用可能な場合)
    """
    contains_confession = detect_confession(model_output)
    confidence_expressed = estimate_expressed_confidence(model_output)

    # キャリブレーション報酬: 内部不確実性と表現された確信度の一致度
    calibration_reward = -abs(
        (1.0 - internal_uncertainty) - confidence_expressed
    )

    # 告白の適切性報酬
    if internal_uncertainty > CONFESSION_THRESHOLD:
        if contains_confession:
            # 不確実な時に告白した: 正の報酬
            confession_reward = 1.0
        else:
            # 不確実な時に告白しなかった: 負の報酬
            confession_reward = -1.0
    else:
        if contains_confession:
            # 確実な時に不要な告白: 軽微な負の報酬
            confession_reward = -0.3
        else:
            # 確実な時に自信を持って回答: 正の報酬
            confession_reward = 0.5

    # 事実性報酬 (正解データがある場合)
    factuality_reward = 0.0
    if ground_truth_available and ground_truth:
        factuality_reward = compute_factuality_score(
            model_output, ground_truth
        )

    return (
        0.4 * calibration_reward
        + 0.4 * confession_reward
        + 0.2 * factuality_reward
    )
🧠 Uncertainty Probe アーキテクチャ 
import torch
import torch.nn as nn


class UncertaintyProbe(nn.Module):
    """
    Transformer の隠れ層活性化から不確実性を推定するプローブ。
    最終層および中間層の表現を入力として、
    トークンレベルの不確実性スコアを出力する。
    """

    def __init__(self, hidden_dim: int, num_layers_to_probe: int = 4):
        super().__init__()
        self.attention_pooling = nn.MultiheadAttention(
            embed_dim=hidden_dim, num_heads=8, batch_first=True
        )
        self.layers = nn.Sequential(
            nn.Linear(hidden_dim * num_layers_to_probe, hidden_dim),
            nn.GELU(),
            nn.Dropout(0.1),
            nn.Linear(hidden_dim, hidden_dim // 2),
            nn.GELU(),
            nn.Linear(hidden_dim // 2, 1),
            nn.Sigmoid(),
        )

    def forward(
        self, hidden_states: list[torch.Tensor]
    ) -> torch.Tensor:
        """
        Args:
            hidden_states: 各層の隠れ状態のリスト
                          [batch_size, seq_len, hidden_dim] x num_layers

        Returns:
            uncertainty_scores: [batch_size, 1] 不確実性スコア
        """
        # 複数層の表現を結合
        concatenated = torch.cat(hidden_states, dim=-1)

        # シーケンス全体のプーリング
        pooled = concatenated.mean(dim=1)

        # 不確実性スコアの予測
        uncertainty = self.layers(pooled)
        return uncertainty

他の誠実性手法との比較

Chain-of-Thought

推論過程を可視化するが、最終的な確信度は明示しない。推論のステップを見せることで透明性を確保するが、ユーザーは結論の信頼度を自分で判断する必要がある。

Self-reflection

回答後に修正を試みるが、不確実性の定量化が困難。事後的な自己チェックであり、不確実性を事前に捉えるメカニズムではない。

Confessions

不確実性を出力レベルで直接表明するため、ユーザーにとって最も解釈しやすい。内部状態の定量的な分析に基づき、適切なタイミングで告白を挿入する。

開発者への影響

AI アプリケーション開発における信頼性設計

  • Confessions メカニズムが API レベルで公開されれば、開発者はモデルの確信度に基づいて UI を動的に変更できる
  • 高リスク領域 (医療、法律、金融) では告白閾値を低く設定し、誤情報の提供リスクを大幅に低減
  • 「モデルが何を知らないか」が明確になるため、ユーザーの全体的な満足度が向上

AI Safety エコシステムへの貢献

  • Confessions が広く採用されれば、AI の誠実性に関する業界標準として機能する可能性
  • EU AI Act などの規制が AI の透明性を要求する中、規制要件を技術的に満たすアプローチ
  • 告白のログを記録することで、モデルの動作を事後的に監査し、安全性の継続的な改善に活用

研究コミュニティへの影響

  • CCE などの新メトリクスがモデル評価の標準的な指標として採用される可能性
  • Honesty Reward の設計パターンは、他の望ましい振る舞い (公平性、安全性) の強化学習にも応用可能
  • テキスト以外のモダリティ (画像、音声) における不確実性告白メカニズムへの発展が期待

まとめ

Confessions は、言語モデルの誠実性 (honesty) を技術的に保証するための新しいアプローチであり、OpenAI の AI Safety 研究における重要な成果である。モデルが自身の不確実性を積極的に「告白」することで、ハルシネーションの影響を軽減し、ユーザーとの信頼関係を構築する。技術的には、Uncertainty Probe と Honesty Reward の組み合わせにより実現され、Chain-of-Thought や Self-reflection と相補的に機能する。特に高リスク領域での AI 活用において重要な基盤技術となり、開発者にとってはアプリケーションにおける信頼性設計の新しいパラダイムを提示するものである。
