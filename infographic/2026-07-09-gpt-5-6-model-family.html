2026-07-09 新機能 / モデル

GPT-5.6 モデルファミリーの発表: Sol / Terra / Luna の 3 モデル構成と新機能群

用途と予算に応じた 3 つのバリエーション構成のフロンティアモデルファミリー。1.05M コンテキスト、128K 出力、Programmatic Tool Calling、マルチエージェントオーケストレーションを搭載。

概要

  • 🌟3 層モデルファミリー: Sol (フラッグシップ)、Terra (バランス型)、Luna (効率型) の 3 つのバリエーションにより、あらゆるユースケースとコスト要件に対応
  • 📐巨大なコンテキスト: 1.05M トークンのコンテキストウィンドウと 128K トークンの最大出力を全モデルで共通提供
  • ⚡Programmatic Tool Calling: モデルが JavaScript を記述・実行してツールを協調制御。隔離された V8 ランタイムで並列ツール実行、ループ、条件分岐を単一リクエスト内で処理
  • 🤖Multi-agent Orchestration (ベータ): ルートエージェントがサブエージェントを並列起動・協調させるマルチエージェント機能を API レベルでサポート
  • 🧠Pro モード推論: 6 段階の推論エフォート (none ~ max) と Pro モードによる深い思考を実行。ターン間での推論の連続性を維持
  • 💾Explicit Prompt Caching Controls: プロンプトキャッシュの動作をきめ細かく制御し、レイテンシとコストを削減
  • 🖼️オリジナル画像サイズ対応: 元の解像度そのままで画像を処理可能 (detail: original / auto)

共通仕様

  • 📏コンテキストウィンドウ: 1.05M トークン (約 105 万トークン)
  • 📤最大出力: 128K トークン
  • 📅知識カットオフ: 2026 年 2 月 16 日
  • 🎚️推論レベル: none, low, medium, high, xhigh, max の 6 段階
  • 🔧ツール: Functions、Web search、File search、Computer use
  • 📥入力: テキストおよび画像
  • 📄出力: テキスト
  • 🔗対応エンドポイント: v1/responses、v1/chat/completions、v1/batch

主要な新機能

⚡
Programmatic Tool Calling
モデルが JavaScript を記述・実行してツールを協調制御。隔離された V8 ランタイムで安全に実行。並列ツール実行、ループ、条件分岐を単一リクエスト内で処理。tools.* 名前空間を通じてツールと対話。Functions、MCP、apply_patch、shell、code_interpreter をサポート。
🤖
Multi-agent Orchestration
Responses API を通じてルートエージェントがサブエージェントを並列起動・協調。6 つのコラボレーションアクション (spawn_agent, send_message, followup_task, wait_agent, interrupt_agent, list_agents)。max_concurrent_subagents パラメータで制御。
🧠
Pro モード推論
reasoning.mode パラメータで standard / pro を選択。Pro モードは特に困難なタスクに対して深い思考を実行。推論コンテキストを current_turn / all_turns で指定し、ターン間での推論の連続性を維持。
💾
Explicit Prompt Caching
プロンプトキャッシュの動作をきめ細かく制御する新しいコントロール。キャッシュのヒット率を最適化し、レイテンシとコストの削減を可能にする。
🖼️
オリジナル画像サイズ対応
画像入力で detail 設定を original または auto に指定することで、元の解像度そのままで画像を処理可能に。高解像度画像の分析精度が向上。
🎚️
6 段階推論エフォート
none, low, medium, high, xhigh, max の 6 段階でタスクの難易度に応じた最適な推論リソースの配分が可能。コストと品質のバランスを細かく制御。

従来モデルとの比較

Before (GPT-5.4 以前)

  • 単一モデル提供のみ
  • 複数ツール協調に複数ラウンドトリップが必要
  • マルチエージェントは独自実装が必須
  • 推論制御は限定的
  • プロンプトキャッシュの細粒度制御なし
  • コンテキストウィンドウが小さい
  • 最大出力トークンに制限
  • 画像入力の解像度制限あり

After (GPT-5.6)

  • Sol / Terra / Luna の 3 層構成で選択可能
  • Programmatic Tool Calling で単一リクエスト内完結
  • マルチエージェントが API レベルでサポート
  • 6 段階推論 + Pro モードで細粒度制御
  • Explicit Prompt Caching Controls 導入
  • 1.05M トークンのコンテキスト
  • 128K トークンの最大出力
  • オリジナル解像度での画像処理が可能

GPT-5.6 Sol の基本利用 (Responses API)
from openai import OpenAI

client = OpenAI()

# GPT-5.6 Sol を使用した基本的な Responses API 呼び出し
response = client.responses.create(
    model="gpt-5.6-sol",
    input=[
        {
            "role": "user",
            "content": "Explain the key differences between microservices and monolithic architectures, including trade-offs for team scaling."
        }
    ],
    reasoning={
        "effort": "high"
    },
    max_output_tokens=4096
)

print(response.output_text)

Multi-agent Orchestration
from openai import OpenAI

client = OpenAI()

# マルチエージェントオーケストレーション: ルートエージェントが
# サブエージェントを並列に起動して協調させる例
response = client.responses.create(
    model="gpt-5.6-sol",
    input=[
        {
            "role": "user",
            "content": "Analyze this codebase: review security vulnerabilities, check performance bottlenecks, and suggest architectural improvements."
        }
    ],
    multi_agent={
        "enabled": True,
        "max_concurrent_subagents": 3
    },
    instructions="""You are a lead architect agent. Spawn specialized subagents for:
1. Security review - analyze code for vulnerabilities
2. Performance analysis - identify bottlenecks
3. Architecture review - suggest structural improvements

Coordinate their findings into a unified report.""",
    tools=[
        {
            "type": "function",
            "name": "read_file",
            "description": "Read a file from the repository",
            "parameters": {
                "type": "object",
                "properties": {
                    "path": {"type": "string", "description": "File path"}
                },
                "required": ["path"]
            }
        }
    ],
    max_output_tokens=16384
)

# レスポンスにはサブエージェントの生成・メッセージ交換・
# 結果統合のプロセスが含まれる
for item in response.output:
    if item.type == "message":
        print(item.content[0].text)
    elif item.type == "agent_action":
        print(f"[Agent Action] {item.action}: {item.agent_id}")
Pro モード推論
from openai import OpenAI

client = OpenAI()

# Pro モード: 困難な数学的推論タスクに対して
# 最大限の推論リソースを投入する例
response = client.responses.create(
    model="gpt-5.6-sol",
    input=[
        {
            "role": "user",
            "content": """Prove that for any prime p > 3,
            p^2 - 1 is divisible by 24.
            Provide a rigorous proof with all intermediate steps."""
        }
    ],
    reasoning={
        "mode": "pro",           # Pro モードで深い推論を実行
        "effort": "max",         # 最大限の推論エフォート
        "context": "all_turns"   # 全ターンの推論コンテキストを維持
    },
    max_output_tokens=8192
)

print(response.output_text)

# 推論トークンの使用状況を確認
print(f"\nInput tokens: {response.usage.input_tokens}")
print(f"Output tokens: {response.usage.output_tokens}")
print(f"Reasoning tokens: {response.usage.reasoning_tokens}")

開発者への影響とユースケース

モデル選択戦略

Sol - 最高精度タスク

複雑な推論、コーディング、専門的分析など最高精度が求められるタスク。プロフェッショナルワーク向けフラッグシップモデル。

Terra - 幅広い用途

一般的なアプリケーション開発、チャットボット、コンテンツ生成など幅広い用途。知能とコストのバランスを最適化。

Luna - コスト効率重視

大量バッチ処理、分類タスク、要約など、コスト効率が重視される場面。最小限のコストで高品質な結果を提供。

主な影響

  • Programmatic Tool Calling による効率化 従来の複数ラウンドトリップが必要なツール協調が、単一リクエスト内で完結。レイテンシの大幅な削減とコスト最適化を実現。
  • マルチエージェント設計パターンの標準化 複雑なワークフローを複数の専門エージェントに分割して並列処理するアーキテクチャが API レベルでサポート。独自のオーケストレーション実装が不要に。
  • 推論の細粒度制御 Pro モードと 6 段階の推論エフォート制御により、タスクの難易度に応じた最適な推論リソースの配分が可能。all_turns コンテキストでマルチターン対話の一貫性を確保。

移行時の考慮事項

  • GPT-5.4 からの移行では新しいモデル ID (gpt-5.6-sol、gpt-5.6-terra、gpt-5.6-luna) への変更が必要
  • Responses API の利用が推奨。新機能の多くは Responses API でのみ利用可能
  • Programmatic Tool Calling は tool_choice: "programmatic" の指定が必要
  • マルチエージェント機能はベータのため、本番利用前に十分なテストを推奨
  • 128K トークンの最大出力は従来モデルと比較して大幅に増加。コスト管理に注意
