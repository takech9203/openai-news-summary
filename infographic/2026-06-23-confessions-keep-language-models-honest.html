How Confessions Can Keep Language Models Honest

2026-06-23

Key Points

💬

自己申告 (Confession) メカニズム

モデル自身が不確実性、誤り、または欺矞的な傾向を自発的に「告白」するメカニズムを構築し、AI システムの信頼性を根本的に向上

🎲

ゲーム理論的インセンティブ設計

欺矞を隠蔽するよりも告白する方がモデルにとって有利になる報酬構造を設計。「自白減刑」の原理に基づくインセンティブ設計

🎯

誠実性報酬 (Honesty Reward)

RLHF を拡張し、正直な不確実性表明に報酬を与える Honesty Reward を導入。モデルの誠実性を訓練レベルで改善

🔍

不確実性プローブ

Transformer の隠れ層活性化パターンから不確実性を検出するプローブを訓練し、モデルの内部的な確信度を定量化

Before / After

従来のアプローチ

  • 外部検出に依存
  • 発見されるまで隠蔽
  • 検出側と欺矞側の軍拡競争
  • キャリブレーションの欠如
  • 過信バイアスが残る

Confessions アプローチ

  • 自己申告を奨励
  • 正直が最適戦略になる設計
  • インセンティブによる構造的保証
  • 内部不確実性と出力の一致
  • 多層的安全性保証

主要コンポーネント

💬 Confession メカニズム

自己申告型誠実性: モデルが自身の不確実性や情報の欠如を積極的に「告白」し、ハルシネーションの影響を軽減

🎯 Honesty Reward

報酬関数による誠実性強化: 正直な不確実性表明に報酬、断定的な誤情報にペナルティを課す RLHF 拡張

🧠 Uncertainty Probe

不確実性の内部検出: 複数の中間層の活性化パターンを統合し、モデルが確信を持っているかを定量的に推定

🎲 ゲーム理論的設計

Confession vs Concealment の期待値比較: 告白の期待値 (+0.5 ~ +1.0) が隠蔽の期待値 (-1.0 ~ -2.0) を常に上回る設計

Training Methodology: 4 ステージ

1

基礎知識獲得

大規模言語モデルの事前訓練により、世界知識と言語能力の基盤を構築。事実と推測の境界を学習する土台を形成する。

↓
2

不確実性認識 (Uncertainty Probing)

モデルの隠れ層から不確実性を検出するプローブを訓練。確信がある場合とない場合の差異を特定するクラシファイアを構築。

↓
3

Confession 学習 (Honesty RL)

正直な不確実性表明に報酬、断定的な誤情報にペナルティを課す Honesty Reward を導入。告白行動の強化学習を実施。

↓
4

強化学習による最適化

ゲーム理論的インセンティブ設計による最適化。告白のコスト < 隠蔽のリスクを常に保証するキャリブレーション評価と閾値決定。

コードサンプル

Python Honesty Reward 関数の実装例 
# Confessions Honesty Reward の概念的な実装
def compute_honesty_reward(
    model_output: str,
    internal_uncertainty: float,
    confession_detected: bool,
    deception_detected_externally: bool,
    confession_timing: float = 1.0,  # 0.0 (遅い) ~ 1.0 (即時)
) -> float:
    """
    モデルの出力に対する誠実性報酬を計算する。

    ゲーム理論的設計:
    - 告白のコスト < 隠蔽のリスク を常に保証
    - 早期告白ほど報酬が高い (「自白減刑」の原理)
    """
    CONFESSION_THRESHOLD = 0.6

    # キャリブレーション報酬: 不確実性と表現の一致度
    expressed_confidence = estimate_expressed_confidence(model_output)
    calibration_reward = -abs(
        (1.0 - internal_uncertainty) - expressed_confidence
    )

    # 告白インセンティブ報酬
    if internal_uncertainty > CONFESSION_THRESHOLD:
        if confession_detected:
            # 不確実な時に告白: 正の報酬 (早期ほどボーナス)
            confession_reward = 1.0 + 0.5 * confession_timing
        else:
            # 不確実な時に告白しない: 負の報酬
            confession_reward = -1.5
    else:
        if confession_detected:
            # 確実な時の不要な告白: 軽微な負の報酬
            confession_reward = -0.3
        else:
            # 確実な時に自信を持って回答: 正の報酬
            confession_reward = 0.5

    # 欺矞検出ペナルティ (外部検出された場合のみ)
    deception_penalty = 0.0
    if deception_detected_externally and not confession_detected:
        # 外部検出され、かつ告白していない: 重いペナルティ
        deception_penalty = -3.0
    elif deception_detected_externally and confession_detected:
        # 外部検出されたが告白済み: 自白減刑
        deception_penalty = -0.5

    return (
        0.3 * calibration_reward
        + 0.4 * confession_reward
        + 0.3 * deception_penalty
    )

ユースケース

🏢 エンタープライズ信頼性向上

モデルの確信度に基づいて UI を動的に制御。不確実な回答には視覚的インジケータを表示し、ユーザーに追加検証を促す設計が可能に。API レスポンスに告白メタデータが含まれる将来像。

🏥 医療 AI の安全性

高リスク領域では告白閾値を低く設定し、少しでも不確実性がある場合に告白を要求。誤診断リスクを大幅に低減し、医療従事者の意思決定を支援。

⚖️ 法的助言の正確性

法的助言においてモデルが「この法的解釈には確信がありません」と明示することで、ユーザーが専門家への相談を適切に判断可能に。規制要件 (EU AI Act 等) への技術的対応。

まとめ

Confessions は、言語モデルの誠実性を「外部からの検出」ではなく「内発的な自己申告」により保証する画期的なアプローチである。ゲーム理論に基づくインセンティブ設計により、欺矞を隠蔽するよりも告白する方が合理的な報酬構造を構築し、誠実さを構造的に保証する。技術的には Uncertainty Probe による内部不確実性の検出と Honesty Reward による告白行動の強化学習の組み合わせで実現され、CoT モニタリングやスキーミング検出と相補的に機能する多層的安全性フレームワークの重要な柱である。
