概要

👤

エンドユーザー別分析

エンドユーザー別のブロック頻度・傾向を分析し、不正利用パターンを早期に検出

⚡

簡単な導入

metadata.safety_identifier の追加のみで利用開始可能。既存の API コールに最小限の変更で対応

コードサンプル

Python Responses API での使用例 
from openai import OpenAI

client = OpenAI()

# Responses API での使用例
response = client.responses.create(
    model="gpt-5.5",
    input="Hello!",
    metadata={
        "safety_identifier": "user_12345"
    }
)
print(response.output_text)
Python Chat Completions API での使用例 
# Chat Completions API での使用例
response = client.chat.completions.create(
    model="gpt-5.5",
    messages=[
        {"role": "user", "content": "Hello!"}
    ],
    metadata={
        "safety_identifier": "user_12345"
    }
)
print(response.choices[0].message.content)

ユースケース

🚨 不正利用の早期検出

特定ユーザーのブロック頻度が急増した場合にアラートを設定し、不正利用を早期に発見・対処

📋 コンプライアンス対応

規制要件に基づくセーフティ対策の実施状況をダッシュボードで証跡として記録・報告

🔧 プロダクト改善

ブロックパターンの分析から、ユーザー体験を損なわない形でのセーフティ設計を最適化

🔔 アラート設定

閾値を超えたブロック発生時に通知を受け取り、プロアクティブな対応を実現

開発者への影響

  • コンプライアンスの強化 — 規制対応のための可視性を大幅に向上
  • 不正利用対策の効率化 — ユーザー別の傾向分析で的確な対策が可能に
  • 運用コストの削減 — 手動モニタリングからダッシュボード監視への移行
  • 実装の簡便さ — metadata に safety_identifier を追加するだけで利用開始
  • プロアクティブなセーフティ管理への移行 — リアクティブからプロアクティブへ
