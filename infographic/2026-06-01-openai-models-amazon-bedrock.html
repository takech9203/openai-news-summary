API 更新 Amazon Bedrock

OpenAI フロンティアモデルと Codex が Amazon Bedrock で利用可能に

2026-06-01 | OpenAI API Changelog / OpenAI News

概要

🚀

GPT-5.5 と Codex が Amazon Bedrock で利用可能

OpenAI のフラッグシップモデル GPT-5.5 および Codex が、Amazon Bedrock プラットフォーム上で正式に提供開始。AWS ネイティブ環境から最先端の AI モデルにアクセス可能に。

🔑

BedrockOpenAI クライアントで簡単に接続

既存の OpenAI SDK インターフェースを維持しつつ、クライアント初期化部分のみの変更で Bedrock 経由に切り替え可能。ビジネスロジックの変更は不要。

🌐

us-east-2 リージョンで初期展開

初期リリースは us-east-2 リージョンで提供。専用エンドポイント bedrock-mantle.us-east-2.api.aws を通じて Responses API にアクセス。

💰

AWS 一括課金でガバナンスを統合

OpenAI モデルの利用料金が AWS の既存請求体系に統合。経理処理の簡素化と、IAM によるチーム/サービス単位のアクセス制御を実現。

Before / After 比較

Before

  • OpenAI API キーでの直接接続
  • 別個の課金管理 (OpenAI 請求)
  • OpenAI 独自のアクセス制御
  • データ処理地域の選択肢が限定的
➡

After

  • AWS IAM 認証でセキュアに接続
  • AWS 一括課金でガバナンス統合
  • IAM ポリシーによる細粒度アクセス制御
  • リージョンコンプライアンスに対応

サポートされる機能

📝

テキスト生成 & ストリーミング

Responses API を通じたテキスト生成とリアルタイムストリーミングに対応

🖼

画像入力 & ファイル入力

マルチモーダル入力として画像やファイルを処理可能

📋

構造化出力 (Structured Outputs)

JSON Schema に準拠した確定的な出力フォーマットを指定可能

🔧

関数呼び出し (Function Calling)

外部ツールやシステムとの連携を可能にする関数呼び出し機能

🧠

推論努力度の指定

Reasoning Effort パラメータでモデルの思考深度を制御

⚡

プロンプトキャッシング

繰り返しプロンプトのキャッシュによるレイテンシ削減とコスト最適化

コードサンプル

Python BedrockOpenAI 基本的な使用方法 
from openai import BedrockOpenAI

client = BedrockOpenAI(aws_region="us-east-2")

response = client.responses.create(
    model="openai.gpt-5.5",
    input="Write a haiku about cloud infrastructure.",
)

print(response.output_text)
Python トークンプロバイダー方式 (IAM 認証) 
from aws_bedrock_token_generator import provide_token
from openai import BedrockOpenAI

client = BedrockOpenAI(
    aws_region="us-east-2",
    bedrock_token_provider=provide_token,
)

response = client.responses.create(
    model="openai.gpt-5.5",
    input="Explain the benefits of serverless architecture.",
)

print(response.output_text)
JavaScript BedrockOpenAI (Node.js / TypeScript) 
import { BedrockOpenAI } from "openai";

const client = new BedrockOpenAI({ awsRegion: "us-east-2" });

const response = await client.responses.create({
    model: "openai.gpt-5.5",
    input: "Write a haiku about cloud infrastructure.",
});

console.log(response.output_text);

想定ユースケース

AWS ネイティブ企業での OpenAI モデル活用

AWS を主要クラウドとして利用する企業が、既存の AWS インフラストラクチャとガバナンスフレームワーク内で OpenAI の最先端モデルを活用可能。経理処理の簡素化と運用の統一を実現する。

IAM による細粒度アクセス制御

AWS IAM ポリシーを使用して、チームやサービスごとにモデルへのアクセス権限を細かく制御。既存の AWS セキュリティ体制との統合により、エンタープライズレベルのアクセス管理を実現する。

データレジデンシー要件への対応

データが特定の AWS リージョン内で処理されるため、規制要件やデータ主権に関するコンプライアンス要件を満たしやすくなる。特に金融、医療、公共セクターでの活用が期待される。
