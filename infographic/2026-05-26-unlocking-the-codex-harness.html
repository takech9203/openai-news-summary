YAML codex-harness-config.yaml 
# codex-harness-config.yaml
harness:
  sandbox:
    type: container
    read_only_root: true
    network_policy: whitelist
    resource_limits:
      cpu: "4"
      memory: "8Gi"
  permissions:
    file_access:
      - path: "/workspace/**"
        mode: read-write
      - path: "/system/**"
        mode: read-only
    commands:
      allow: ["git *", "npm *", "cargo *"]
      deny: ["rm -rf /", "curl * | sh"]
  hooks:
    on-turn-start:
      command: "python /hooks/validate_context.py"
    on-thread-idle:
      command: "python /hooks/cleanup_temp.py"
    on-tool-call:
      command: "python /hooks/audit_tool_use.py"
Python /hooks/audit_tool_use.py 
# /hooks/audit_tool_use.py
import json, sys
from datetime import datetime

def audit_tool_call():
    hook_input = json.loads(sys.stdin.read())
    log_entry = {
        "timestamp": datetime.utcnow().isoformat(),
        "tool": hook_input.get("tool_name"),
        "subagent": hook_input.get("subagent_identity"),
        "action": "allowed"
    }
    with open("/var/log/codex/audit.jsonl", "a") as f:
        f.write(json.dumps(log_entry) + "\n")
    sys.exit(0)

if __name__ == "__main__":
    audit_tool_call()

ユースケース

🏢 エンタープライズ開発チーム

プロファイルで環境分離し、本番環境にはコンプライアンスフックを付与。チーム全体で統一された安全な開発体験を実現

🔒 セキュリティ重視の開発

サンドボックスとパーミッション制御で攻撃面を最小化。デフォルト拒否ポリシーにより、明示的に許可されたリソースのみアクセス可能

🔧 MCP ツール統合

社内ドキュメント検索やテストランナーをエージェントに接続。OAuth 認証で安全にツールエコシステムを拡張

開発者への影響

  • フックによるカスタムロジック注入 (CI/CD、セキュリティスキャン、コンプライアンスチェック)
  • プロファイルベースの環境管理 (開発/本番の一括切替)
  • MCP サーバーによるツールエコシステム拡張
  • デフォルト拒否ポリシーに基づく安全な実行環境
  • codex doctor による診断とデバッグ改善
