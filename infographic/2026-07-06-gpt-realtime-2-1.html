GPT-Realtime-2.1 および GPT-Realtime-2.1 mini のリリース

📅 2026-07-06

概要

🎯

リアルタイム音声対話の新時代

英数字認識、無音・ノイズ処理、割り込み動作が大幅に改善された次世代リアルタイム推論モデル

⚡

2 つのモデルバリエーション

最高レベルの推論能力を持つフルバージョンと、高速・低コストを実現する mini バージョンを提供

🎨

マルチモーダル対応

テキスト、音声、画像の入力に対応し、複雑な音声エージェントワークフローに最適化

💰

コスト効率の向上

mini モデルでテキスト入力 85% 削減、音声入力 68.75% 削減、出力コスト 90% 削減を実現

GPT-Realtime-2 からの改善点

Before (GPT-Realtime-2)

  • ❌ 英数字認識の精度に課題
  • ❌ ノイズ環境での処理が不安定
  • ❌ 割り込み応答の遅延
  • ❌ 固定された推論レベル

After (GPT-Realtime-2.1)

  • ✅ 英数字認識が大幅に向上
  • ✅ 無音・ノイズ処理の改善
  • ✅ 割り込み動作の最適化
  • ✅ 設定可能な推論努力

主な特徴

GPT-Realtime-2.1

🏆 最高レベルの推論

128,000 トークンのコンテキストウィンドウ、32,000 トークンの最大出力、設定可能な推論努力により、複雑なタスクに対応

GPT-Realtime-2.1 mini

⚡ 高速・低コスト

蒸留された軽量モデルで、非常に高速な応答速度とコスト効率を実現。大量の音声対話に最適

🔗 関数呼び出し

ツール呼び出しをサポートし、外部システムや MCP との統合が容易。天気取得、データベースアクセスなどが可能

🎙️ 多様な接続方式

WebRTC (ブラウザ)、WebSocket (サーバー)、SIP (テレフォニー) の 3 つの接続方式をサポート

🔊 音声検出 (VAD)

Server VAD により、ユーザーの発話開始・終了を自動検出。自然な会話フローを実現

📝 リアルタイム文字起こし

音声をリアルタイムでテキスト化。Whisper-1 モデルによる高精度な文字起こしをサポート

コードサンプル

WebSocket 接続による基本的な音声対話の実装例

Python GPT-Realtime-2.1 mini を使用したリアルタイム対話 
import asyncio
import websockets
import json

async def realtime_mini_chat():
    """GPT-Realtime-2.1 mini による高速・低コスト対話"""

    url = "wss://api.openai.com/v1/realtime"
    headers = {
        "Authorization": f"Bearer {YOUR_API_KEY}",
        "OpenAI-Beta": "realtime=v1"
    }

    async with websockets.connect(url, extra_headers=headers) as ws:
        # mini モデルを使用
        session_config = {
            "type": "session.update",
            "session": {
                "model": "gpt-realtime-2.1-mini",
                "modalities": ["text", "audio"],
                "instructions": "簡潔に応答してください。",
                "voice": "alloy",
                "turn_detection": {
                    "type": "server_vad",
                    "threshold": 0.5
                }
            }
        }
        await ws.send(json.dumps(session_config))

        # テキストメッセージを送信
        text_message = {
            "type": "conversation.item.create",
            "item": {
                "type": "message",
                "role": "user",
                "content": [
                    {
                        "type": "input_text",
                        "text": "今日の天気を教えてください。"
                    }
                ]
            }
        }
        await ws.send(json.dumps(text_message))

        # 応答生成をトリガー
        create_response = {
            "type": "response.create"
        }
        await ws.send(json.dumps(create_response))

        # 応答を受信
        async for response in ws:
            data = json.loads(response)
            if data["type"] == "response.text.done":
                print(f"応答: {data['text']}")
                break

asyncio.run(realtime_mini_chat())

ユースケース

🏥 医療相談・カスタマーサポート

長いコンテキストを保持し、専門知識を活用した高精度な対話が必要な場面で GPT-Realtime-2.1 を使用。電話番号、住所、カルテ番号などの英数字情報を正確に認識します。

📞 大規模コールセンター・IVR システム

GPT-Realtime-2.1 mini により、大量の音声対話をコスト効率よく処理。簡単な質問応答や案内システムに最適で、低遅延とコスト削減を同時に実現します。

🌍 リアルタイム音声翻訳

ノイズ耐性の向上により、屋外や騒がしい環境でも高品質な音声翻訳が可能。観光案内、国際会議、フィールドサービスなどで活躍します。

🤖 音声エージェント・アシスタント

関数呼び出しによるツール統合で、天気取得、予約管理、データベース検索などを実行。割り込み動作の最適化により、自然な対話体験を提供します。

まとめ

GPT-Realtime-2.1 および GPT-Realtime-2.1 mini は、リアルタイム音声対話アプリケーションにおける重要な進化を示しています。英数字認識、ノイズ処理、割り込み動作の改善により、より自然で信頼性の高い音声エージェントの構築が可能になります。

フルバージョンの GPT-Realtime-2.1 は、複雑な推論と高い精度が求められるアプリケーションに最適であり、128,000 トークンの大規模なコンテキストウィンドウと設定可能な推論努力により、専門的なドメインでの使用にも対応できます。

一方、GPT-Realtime-2.1 mini は、コストと速度を最適化した軽量版として、大規模な音声対話システムやコスト効率が重要なアプリケーションに理想的な選択肢です。テキスト入力で 85% のコスト削減を実現しながら、高レベルの推論能力と同等のマルチモーダル機能を維持しています。
