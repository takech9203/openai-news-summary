GPT-5.5 System Card

OpenAI 史上最も知的なモデルの安全性評価と透明性文書

2026-04-23

Key Points

🛡

安全性評価

Preparedness Framework に基づき、サイバーセキュリティ、CBRN、説得・操作、自律性、エージェント安全性の 5 カテゴリでリスクを体系的に評価。すべてのカテゴリで High 未満のデプロイ基準を充足。

🔍

レッドチーミング

内部安全性チームと外部専門家による包括的なレッドチーミングを実施。ジェイルブレイク耐性、CBRN 知識の引き出し、エージェント行動の安全性を網羅的にテスト。Bio Bug Bounty による継続的検証も開始。

⚖

アライメント

RLHF、Model Spec 準拠訓練、安全性ファインチューニングによる多層的アライメント。人間の意図・価値観への整合と安全性制約の両立を追求。

🤖

エージェント安全性

Codex プラットフォームとの統合による自律的タスク実行のリスクを新規評価項目として追加。サンドボックス実行、権限の最小化、確認ステップの挿入、監査ログによる多層的な安全対策。

📋

Preparedness Framework

OpenAI のフロンティアモデル向けリスク評価枠組み。Low / Medium / High / Critical の 4 段階でリスクを定量的に評価し、デプロイ可否を判定。

概要

OpenAI は 2026 年 4 月 23 日、最新フラッグシップモデル「GPT-5.5」の公開と同時に、同モデルの安全性評価と技術詳細を包括的にまとめた System Card を公開した。System Card は、モデルの能力評価、安全性テストの結果、特定されたリスク領域、アライメント手法、既知の制限事項、およびデプロイに際して講じられた安全対策を体系的に記述した透明性文書である。

GPT-5.5 は「史上最も知的なモデル (our smartest model yet)」と位置づけられるフラッグシップモデルであり、コーディング、リサーチ、データ分析といった複雑なタスクにおいて、複数のツールを横断的に活用しながら高度な処理を実行する。本 System Card は GPT-5.3 Instant、GPT-5.4 Thinking、ChatGPT Images 2.0 に続く System Card シリーズの最新版であり、同日公開の GPT-5.5 Bio Bug Bounty プログラムとも密接に連動している。

注記: 本レポートは RSS フィード情報、同日公開の GPT-5.5 紹介記事および Bio Bug Bounty の情報、過去の System Card の構造と内容、ならびに OpenAI の安全性関連の公開情報に基づいて作成されている。正確な詳細については 公式ページ を参照されたい。

System Card の位置づけと目的

📖
透明性の確保: モデルの能力と限界を正確に開示し、利用者が情報に基づいた判断を下せるようにする
✅
安全性の説明責任: 安全性評価の手法と結果を公開し、外部からの検証を可能にする
🏭
業界標準の形成: 安全性評価の枠組みを公開し、AI 業界全体の安全基準の向上に寄与する
🏛
政策立案への貢献: 規制当局や政策立案者に AI モデルのリスクプロファイルに関する客観的情報を提供する

5 つのリスクカテゴリ

Preparedness Framework に基づき、GPT-5.5 の安全性は以下の 5 つの主要リスクカテゴリについて評価されている。

💻 サイバーセキュリティ

強化されたコーディング能力に伴うサイバー攻撃への悪用可能性を評価。

  • 脆弱性の発見と悪用
  • マルウェア生成
  • フィッシング・ソーシャルエンジニアリング
  • 攻撃の自動化

☣ CBRN (生物・化学・放射線・核)

大量破壊兵器に関連する危険な知識の生成可能性を評価。Bio Bug Bounty と連動。

  • 生物兵器の製造知識
  • 化学兵器の合成経路
  • 核兵器関連知識
  • デュアルユース研究の悪用

🗣 説得・操作

大規模な世論操作やソーシャルエンジニアリングへの悪用可能性を評価。

  • 偽情報の生成
  • 大規模な影響力工作
  • パーソナライズされた説得
  • 信頼性の偽装

🎯 自律性

モデルが人間の監督を逃れて自律的に行動する可能性を評価。マルチツール統合により一層厳格に実施。

  • タスク逸脱
  • リソース獲得
  • 自己保存行動
  • 目標の一般化

🌐 マルチモーダル

画像や音声を含むマルチモーダル入出力における安全性リスクを評価。

  • 有害な画像生成の誘発
  • 画像内の危険情報の解読
  • クロスモーダル攻撃耐性

アライメント手法

🧠 RLHF (Reinforcement Learning from Human Feedback)

人間の評価者からのフィードバックに基づく強化学習。GPT-5.5 アライメントの基盤技術。

  • 人間の選好データに基づく報酬モデルの訓練
  • PPO (Proximal Policy Optimization) によるポリシー最適化
  • 複数ラウンドの反復的改善で安全性と有用性のバランスを調整

📜 Model Spec 準拠

2026 年 3 月 25 日公開の「Our Approach to the Model Spec」に基づく明示的なモデル仕様への準拠訓練。

  • モデルの望ましい行動規範の形式的定義
  • 安全性制約とコンテンツポリシーの組み込み
  • ユーザーインタラクションガイドラインの遵守

📊 安全性訓練データの拡充

多様なデータソースを活用した安全性ファインチューニング。

  • レッドチーミングデータ (攻撃と防御のペア)
  • Safety Bug Bounty プログラムで報告された脆弱性データ
  • AI 生成の合成安全データ
  • ドメイン専門家のアノテーション (CBRN、サイバーセキュリティ等)

レッドチーミングの実施

🔒 内部レッドチーミング

OpenAI 内部安全性チームによる網羅的なテスト。

  • 安全ガードレールの突破テスト
  • CBRN 知識の引き出しテスト
  • エージェント行動の安全性テスト
  • マルチターン攻撃テスト

🌍 外部レッドチーミング

外部専門家チームによる多様な攻撃手法と視点の導入。

  • ドメイン専門家 (生物学、化学、サイバーセキュリティ)
  • セキュリティ研究者 (プロンプトインジェクション、ジェイルブレイク)
  • 政策・倫理の専門家 (社会的影響、バイアス)
Bio Bug Bounty: 外部レッドチーミングを継続的なプログラムとして制度化。最大報奨金 $25,000。System Card の静的な評価を動的に補完する仕組みとして位置づけられている。

エージェント安全性 (Agentic Safety)

GPT-5.5 が Codex プラットフォームと統合され、自律的なコード実行やツール呼び出しを行う能力を持つことから、エージェント安全性は本 System Card における重要な新規評価項目である。

エージェント的行動におけるリスク

⚠
権限昇格: 設計上の権限範囲を超えたアクション実行のリスク
⚠
意図しないツール呼び出し: 悪意のある入力による不正なツール操作のリスク
⚠
チェーン攻撃: 複数のツール呼び出しを連鎖させた安全ガードレールの迂回リスク
⚠
環境への不可逆的変更: ファイルシステムやデータベースへの不可逆的変更リスク
⚠
プロンプトインジェクション: 外部データソースの悪意あるプロンプトによるエージェント乗っ取りリスク

安全対策

🛡
サンドボックス実行: Codex 環境におけるコード実行のサンドボックス化
🔑
権限の最小化: 最小権限原則の適用
✔
確認ステップの挿入: 高リスクアクション実行前にユーザーの明示的承認を要求
📝
監査ログ: すべてのアクションを記録し事後検証を可能に

デプロイに際する安全対策

GPT-5.5 のデプロイに際して講じられた多層的な安全対策。

🚫
入力フィルタリング: 危険なリクエストを処理前にブロック
🔍
出力フィルタリング: 生成応答内の危険コンテンツを検証
⏱
レート制限: 悪意のある大量リクエストを防止
📜
使用ポリシー: API 利用規約に基づく不正利用の検出・防止
📡
継続的モニタリング: リアルタイム監視と異常検知
🚨
インシデント対応: 問題発見時の迅速な対応体制
📈
段階的ロールアウト: リスク低減のための段階的公開

コードサンプル

Python GPT-5.5 の安全性パラメータを活用した呼び出し例 
from openai import OpenAI

client = OpenAI()

# GPT-5.5 の安全性パラメータを活用した呼び出し例
response = client.chat.completions.create(
    model="gpt-5.5",
    messages=[
        {
            "role": "system",
            "content": (
                "You are a helpful research assistant. "
                "Adhere strictly to factual information and cite sources. "
                "Decline requests for harmful content."
            )
        },
        {
            "role": "user",
            "content": "Analyze the latest trends in renewable energy policy."
        }
    ],
    max_completion_tokens=4096,
    # 安全性に関連する推論制御
    reasoning_effort="high"
)

print(response.choices[0].message.content)
Python Moderation API による追加の安全性検証 
from openai import OpenAI

client = OpenAI()

# GPT-5.5 の応答に対するモデレーションチェック
response = client.chat.completions.create(
    model="gpt-5.5",
    messages=[
        {"role": "user", "content": "ユーザーからの質問"}
    ]
)

output_text = response.choices[0].message.content

# Moderation API による追加の安全性検証
moderation = client.moderations.create(
    model="omni-moderation-latest",
    input=output_text
)

if moderation.results[0].flagged:
    print("安全性フラグが検出されました")
    for category, flagged in moderation.results[0].categories:
        if flagged:
            print(f"  - カテゴリ: {category}")
else:
    print("安全性チェック通過")
    print(output_text)
Python Responses API を使用した安全なエージェント構築例 
from openai import OpenAI

client = OpenAI()

# Responses API を使用した安全なエージェント構築例
response = client.responses.create(
    model="gpt-5.5",
    input="Analyze the repository structure and suggest security improvements.",
    tools=[
        {"type": "code_interpreter"},
        {"type": "file_search"},
        {"type": "web_search_preview"}
    ],
    # エージェントの安全性に関する指示を明示的に含める
    instructions=(
        "You are a security-focused code reviewer. "
        "Never execute destructive operations. "
        "Always explain proposed changes before making them. "
        "Request user confirmation for any file modifications."
    )
)

print(response.output_text)

開発者への影響

🤖 エージェント安全設計

GPT-5.5 のエージェント能力を活用する開発者は、System Card で指摘されたリスクを考慮した設計が求められる。

  • 最小権限の原則: エージェントに付与するツールアクセス権限を必要最小限に制限
  • 確認ステップの実装: 高リスクなアクション (ファイル削除、外部 API 呼び出し、データ送信等) 前にユーザーの明示的承認を要求
  • サンドボックスの活用: Codex 環境のサンドボックス機能を活用し実行環境を隔離
  • 監査ログの実装: エージェントのすべてのアクションを記録し事後検証と問題追跡を可能に

🛡 プロンプトインジェクション対策

GPT-5.5 の強化された能力はプロンプトインジェクション攻撃のリスクをも増大させる。

  • Instruction Hierarchy の活用: システムプロンプトとユーザー入力の明確な分離
  • 入力のサニタイズ: 外部データソースからの入力に対する前処理とフィルタリング

既知の制限事項

GPT-5.5 System Card で開示されていると考えられる既知の制限事項。

  • ハルシネーション (幻覚): 事実と異なる情報を自信を持って生成するリスクは完全には排除されておらず、特に専門的な領域や最新の情報に関して発生する可能性がある
  • 知識のカットオフ: トレーニングデータの期限以降の情報については正確な回答が保証されない
  • 推論の不完全性: 複雑な多段階推論において、中間ステップで誤りが蓄積する可能性がある
  • バイアス: トレーニングデータに含まれる社会的・文化的バイアスが出力に反映される可能性がある
  • 過度の従順性 (Sycophancy): ユーザーの期待に過度に合わせようとして、正確性を犠牲にする傾向がある場合がある
  • エージェント行動の不確実性: マルチツール統合による自律的なタスク実行において、予期しない動作や非効率な手順が発生する可能性がある
  • コンテキスト長の実効性: 長大なコンテキストウィンドウをサポートしていても、コンテキストの末尾や中間部分の情報に対する注意力が低下する「Lost in the Middle」問題が残存する可能性がある
