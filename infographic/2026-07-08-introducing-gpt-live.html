GPT-Live の発表: 自然な人間 - AI 対話のための次世代音声モデル

2026 年 7 月 8 日 | OpenAI News/Blog
人間同士の会話と区別がつかないほど自然な対話を実現する次世代モデル
概要ポイント
  • 🎙️ 予測型ターンテイキング: 相手の発話終了を正確に検知し、自然なタイミングで応答を開始。不自然な間や早すぎる割り込みを大幅に削減
  • 💡 感情検出と表現: ユーザーの声のトーン・テンポ・抑揚から感情状態を推定し、適切な感情表現を含む応答を生成
  • 🔄 適応的会話スタイル: 会話の文脈やユーザーの話し方に応じて、フォーマルからカジュアルまで応答スタイルを動的に調整
  • ⚡ 超低遅延: 200ms 以下の応答遅延を実現し、人間の対話と同等の応答速度で会話の流れを妨げない
  • 👁️ マルチモーダル統合: カメラや画面共有からの視覚入力と音声を組み合わせたリアルタイム対話が可能
主要機能
🧠

予測型ターンテイキング

発話終了を事前に予測し、人間同士の会話に匹敵する自然な話者交替を実現

😊

感情検出

多次元の感情分析により、声のトーンや抑揚からユーザーの感情を高精度に推定

🎨

適応的スタイル

文脈に応じてフォーマル / カジュアルを動的に切り替え、自然な会話体験を提供

💬

バックチャネル

「うん」「なるほど」などの相づちを適切なタイミングで挿入し、対話の自然さを向上

🚀

超低遅延応答

200ms 以下のレスポンスタイムで、リアルタイムの会話フローを途切れさせない

モデル比較: GPT-Realtime-2.1 vs GPT-Live

GPT-Realtime-2.1 (従来)

  • ⏱️ 低遅延
  • 🔄 改善済みターンテイキング
  • 😐 基本的な感情認識
  • 🔇 限定的な感情表現
  • 📝 テキスト + 音声 + 画像入力
  • 📦 128K トークン
  • ❌ バックチャネルなし
  • 🔒 固定スタイル

GPT-Live (次世代)

  • ⚡ 超低遅延 (人間同等)
  • 🧠 予測型ターンテイキング
  • 🎭 高度な多次元感情認識
  • 😊 自然な感情表現
  • 👁️ 音声 + ビジョンのリアルタイム統合
  • 📦 拡張 (長時間対話最適化)
  • 💬 適応的バックチャネル
  • 🎨 動的スタイル適応

GPT-Live を使用したリアルタイム音声セッションの作成例

PYTHON
from openai import OpenAI

client = OpenAI()

# GPT-Live を使用したリアルタイム音声セッションの作成
session = client.realtime.sessions.create(
    model="gpt-live",
    modalities=["text", "audio"],
    voice="alloy",
    turn_detection={
        "type": "predictive",  # 予測型ターンテイキング
        "threshold": 0.6,
        "silence_duration_ms": 300,
    },
    emotion_detection=True,
    adaptive_style=True,
)

print(f"Session ID: {session.id}")
print(f"Model: {session.model}")

ユースケース
🎧

カスタマーサポート

感情検出と適応スタイルにより、顧客の状態に合わせた共感的な対応を自動化

教育ツール

バックチャネルと低遅延により、個別指導に匹敵する自然な学習対話を実現

開発者への影響
  • 🔧 API 移行: モデル ID を gpt-live に変更するだけで新モデルへの移行が可能。新イベントタイプ (感情検出、バックチャネル) への対応が必要
  • 💰 コスト考慮: 高度な機能のため GPT-Realtime-2.1 より単価が高い可能性。ユースケースに応じたモデル選択が重要
  • 🔀 使い分け: 基本的な音声対話には GPT-Realtime-2.1 (コスト効率)、高度な自然対話には GPT-Live を選択
