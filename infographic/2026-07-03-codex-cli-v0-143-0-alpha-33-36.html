Before / After

Before

  • Half-open WebSocket 接続が OS TCP タイムアウトまで残存
  • マルチエージェント通信が複数経路で送信
  • TTFT やツール呼び出し時間の計測手段なし
  • インストーラーが 4 回の GitHub API コール
  • 添付ファイルが全て text/plain に誤分類

After

  • 60 秒 Pong 要求で half-open 接続を自動検出・クリーンアップ
  • 単一メソッドに統合 + 構造化ログで可観測性確保
  • ttft_ms / codex.tool_call で詳細なパフォーマンス分析
  • メタデータ再利用で 1 回に最適化
  • ファイル名から正しい MIME タイプを自動検出

セキュリティ修正

🛡️

RUSTSEC-2026-0194 (DoS)

quick-xml クレートの脆弱性。特定の XML 入力によりプロセスのリソースが枯渇する可能性。quick-xml 0.41.0 へのアップグレードで修正。

🛡️

RUSTSEC-2026-0195 (DoS)

関連する quick-xml の DoS 脆弱性。同じく 0.41.0 へのアップグレードで対応。信頼済みパス (plist, wayland-scanner) には一時的な例外を設定。

バグ修正

🐛

Bedrock モデルの可用性メタデータ (PR #30897)

GPT-5.6 Bedrock 上で GPT-5.5 のローンチコピーが表示される問題を解消。availability_nux と upgrade フィールドをクリア。

🐛

インクリメンタル WebSocket リクエスト (PR #30770)

メタデータを無視してコンテンツのみ比較することで、インクリメンタルリクエストの成功率が向上。

🐛

MIME タイプの誤分類 (PR #30796)

パスベースのフィードバック添付ファイルが全て text/plain とラベルされていた問題を修正。gzip → application/gzip、不明 → application/octet-stream に正しく分類。

🐛

GitHub Release メタデータ再利用 (PR #31056)

API コール数を 4 → 1 に削減。レート制限エラーを正しく報告し、mocked-curl 回帰テストを追加。

開発者への影響

📊 可観測性の向上によるデバッグ効率化

TTFT メトリクスとツール呼び出しタイミングの追加により、パフォーマンス問題の特定が格段に容易に。特に ttft_ms は、モデルのレスポンス遅延とネットワークレイテンシを切り分けるための重要な指標となる。

🔌 WebSocket 接続の信頼性向上

Liveness bounds の導入により、half-open 接続が自動検出・クリーンアップされるため、長時間実行タスクでの接続切断問題が軽減される。

🏢 エンタープライズ向けマルチエージェント設定

multi_agent_mode_hint_text の設定オプション追加により、組織固有のガイドラインやポリシーをマルチエージェントモードに反映可能に。

🛡️ セキュリティ脆弱性の修正

quick-xml の DoS 脆弱性 (RUSTSEC-2026-0194/0195) が修正されたため、セキュリティ監査の観点からアップデートが推奨される。

⚡ インストーラー体験の改善

GitHub API コール数の 4 回 → 1 回への削減により、レート制限に遭遇するリスクが低下し、CI/CD パイプラインでの自動インストールがより安定する。

☁️ Bedrock モデル表示の修正

GPT-5.6 Bedrock で GPT-5.5 のローンチコピーが表示される問題が解消。AWS Bedrock 経由で Codex を利用しているユーザーに直接影響する修正。
