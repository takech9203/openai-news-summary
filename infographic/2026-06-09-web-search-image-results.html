API 更新

Web Search で画像検索結果が返却可能に

2026-06-09

キーポイント

🖼

画像検索結果の新規追加

Web Search ツールが image_result タイプを返却可能に。テキストと画像を同時に取得して包括的な検索体験を実現

🔍

シンプルな有効化

search_content_types パラメータに "image" を追加するだけで利用可能。既存実装への影響は最小限

🛒

豊富なユースケース

商品写真、ランドマーク、イベント、ビジュアルリファレンスなど視覚的情報が重要なシーンで活用

🏞

リッチなレスポンス構造

画像 URL、サムネイル、キャプション、ソースページ URL を含む構造化データを返却

⚙

細かな制御が可能

image_settings で max_results と caption を制御。コストと品質のバランスを最適化

Before / After

Before (従来)

  • 検索結果はテキストのみ
  • ビジュアル情報は非対応
  • コンテンツタイプの指定なし
  • 画像設定パラメータなし

After (今回のアップデート)

  • テキスト + 画像の同時取得
  • image_result で画像 URL / サムネイル / キャプション取得可能
  • search_content_types で選択
  • image_settings で結果数 / キャプション制御

主なユースケース

🛍 商品写真

EC サイトやレビューアプリで、商品の外観を検索結果として表示。ビジュアルで購買判断をサポート

🗼 ランドマーク

旅行アプリや地図サービスで、観光地のビジュアルを提供。実際の風景を事前に確認可能

📰 イベント

ニュースやイベント情報に関連する画像を自動取得。リアルタイムな視覚情報を提供

📚 ビジュアルリファレンス

技術ドキュメントやチュートリアルに視覚的な補足情報を追加。理解度を向上

新しいパラメータ

パラメータ 型 説明
search_content_types array "text" / "image" を配列で指定
image_settings.max_results number 取得する画像結果の最大数
image_settings.caption boolean true で画像にキャプションを付与
include array "web_search_call.results" で画像結果を直接参照

レスポンスの image_result フィールド

フィールド 説明
type "image_result" (固定値)
image_url 画像の正規 URL
source_website_url 画像が掲載されているソースページの URL
thumbnail_url サムネイル URL (利用可能な場合)
caption 画像のキャプション (利用可能な場合)

コードサンプル

Python Web Search with Image Results 
from openai import OpenAI

client = OpenAI()

response = client.responses.create(
    model="gpt-5.5",
    tools=[
        {
            "type": "web_search",
            "search_content_types": ["image", "text"],
            "image_settings": {
                "max_results": 3,
                "caption": True,
            },
        }
    ],
    include=["web_search_call.results"],
    input="Show me photos of the Eiffel Tower at night",
)

# レスポンスから画像結果を取得
for item in response.output:
    if item.type == "web_search_call":
        for result in item.results:
            if result.type == "image_result":
                print(f"Image: {result.image_url}")
                print(f"Caption: {result.caption}")
                print(f"Source: {result.source_website_url}")
                print("---")

開発者への影響

マルチモーダル検索体験の構築が容易に

テキストと画像を組み合わせた検索アプリケーションを、単一の API コールで実現できるようになった

既存実装への影響が最小限

search_content_types を追加するだけで画像検索を有効化でき、既存のテキスト検索実装を壊さない

レスポンス解析の変更が必要

画像結果は web_search_call.results に格納されるため、レスポンスのパース処理を更新する必要がある

UI/UX の拡張機会

画像サムネイルやキャプションを活用した、よりリッチなユーザーインターフェースを構築可能

コスト考慮

画像検索結果の取得が API 利用量に影響する可能性があるため、max_results を適切に設定することを推奨
