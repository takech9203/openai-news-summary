API 更新

Custom Voices API サポートの追加

2026-03-13

概要

🎤

カスタムボイス対応

Python SDK v2.28.0 および Node SDK v6.29.0 で Custom Voices API のサポートが追加。独自の音声 ID を指定して音声生成が可能に。

🔗

3 つの API エンドポイント対応

Audio Speech API、Chat Completions Audio、Realtime API の 3 つのエンドポイントでカスタムボイスが利用可能。

🔄

後方互換性の維持

既存のビルトインボイス (alloy、ash、ballad、coral、echo、sage、shimmer、verse、marin、cedar) はそのまま動作。新しい VoiceID オブジェクト形式が追加。

🏢

ブランドボイスの構築

企業やブランドが独自の音声アイデンティティを構築するための基盤。カスタマーサポート、ナレーション、音声アシスタントなどの用途に対応。

Before / After

Before

ビルトインボイスのみ

voice パラメータには文字列リテラル (例: "alloy") のみ指定可能。10 種類の固定ボイスに限定。

voice = "alloy"
➡

After

カスタムボイス対応

VoiceID オブジェクトで独自の音声 ID を指定可能に。ビルトインとカスタムの両方をサポート。

voice = {"id": "voice_1234"}

対応 API エンドポイント

🔊 Audio Speech API

テキストから音声を生成する TTS エンドポイント。gpt-4o-mini-tts モデルを使用し、カスタムボイスでの音声合成が可能。

💬 Chat Completions Audio

チャット補完の音声出力機能。gpt-4o-audio-preview モデルでテキストと音声の両方のモダリティを出力可能。

⚡ Realtime API

リアルタイム音声対話機能。セッション中にモデルが一度音声で応答した後はボイスの変更不可に注意。

技術詳細

カスタムボイスを参照するための新しい型 VoiceID が導入され、Voice 型は TypeAlias として拡張された。

VoiceID 型

id フィールド (Required[str]) を持つ TypedDict。カスタムボイス ID (例: voice_1234) を指定。

Voice TypeAlias

Union[str, Literal["alloy", ...], VoiceID] として定義。ビルトインとカスタムの両方を受容。

VoiceID 型定義 (Python)
class VoiceID(TypedDict, total=False):
    """Custom voice reference."""

    id: Required[str]
    """The custom voice ID, e.g. `voice_1234`."""
Voice TypeAlias (Python)
Voice: TypeAlias = Union[
    str,
    Literal["alloy", "ash", "ballad", "coral", "echo", "sage", "shimmer", "verse", "marin", "cedar"],
    VoiceID
]

コードサンプル

🐍 Audio Speech API (Python)
from openai import OpenAI

client = OpenAI()

# ビルトインボイスを使用
response = client.audio.speech.create(
    model="gpt-4o-mini-tts",
    input="Hello, this is a test of text to speech.",
    voice="marin"
)

# カスタムボイスを使用
response = client.audio.speech.create(
    model="gpt-4o-mini-tts",
    input="Hello, this is a test with a custom voice.",
    voice={"id": "voice_1234"},
    instructions="Speak in a warm, professional tone."
)

response.stream_to_file("output.mp3")
🐍 Chat Completions Audio (Python)
from openai import OpenAI

client = OpenAI()

response = client.chat.completions.create(
    model="gpt-4o-audio-preview",
    modalities=["text", "audio"],
    audio={
        "voice": {"id": "voice_1234"},
        "format": "mp3"
    },
    messages=[
        {
            "role": "user",
            "content": "Tell me about the weather today."
        }
    ]
)

print(response.choices[0].message.content)
🟢 Audio Speech API (Node.js)
import OpenAI from "openai";

const client = new OpenAI();

// Audio Speech API でカスタムボイスを使用
const response = await client.audio.speech.create({
    model: "gpt-4o-mini-tts",
    input: "Hello, this is a test with a custom voice.",
    voice: { id: "voice_1234" },
    instructions: "Speak in a warm, professional tone."
});

const buffer = Buffer.from(await response.arrayBuffer());
await fs.promises.writeFile("output.mp3", buffer);

開発者への影響

後方互換性

既存のビルトインボイス名を文字列で指定するコードはそのまま動作する。破壊的変更はなく、既存のアプリケーションに影響はない。

型定義の変更

TypeScript や Python の型チェックを使用している場合、voice パラメータの型が拡張されたことに注意が必要。Voice 型は文字列リテラル、ビルトインボイス名、VoiceID オブジェクトの Union 型となった。

SDK アップグレード推奨

  • Python SDK: v2.28.0 以上へのアップグレードが必要
  • Node SDK: v6.29.0 以上へのアップグレードが必要

Realtime API での注意事項

Realtime API でカスタムボイスを使用する場合、セッション中にモデルが一度音声で応答した後はボイスの変更ができない。推奨ボイスとして marin と cedar が引き続き最高品質として挙げられている。

SDK 更新内容 (Python SDK v2.28.0)

audio/speech.py

voice パラメータの型を Voice 型に変更

speech_create_params.py

VoiceID クラスと Voice TypeAlias を追加

chat_completion_audio_param.py

Chat Completions 用の VoiceID と Voice を追加

realtime_audio_config_output.py

Realtime API 出力設定にカスタムボイス対応を追加

realtime_response_create_audio_output.py

Realtime レスポンス作成にカスタムボイス対応を追加

Node SDK v6.29.0 でも同様の変更が適用されている。

まとめ

今回の SDK アップデートにより、OpenAI の音声関連 API 全体でカスタムボイスがサポートされた。VoiceID オブジェクト ({ "id": "voice_1234" }) を voice パラメータに指定するだけで、Audio Speech API、Chat Completions Audio、Realtime API の 3 つのエンドポイントでカスタムボイスを利用できる。既存のビルトインボイスとの後方互換性を維持しつつ、企業やブランドが独自の音声アイデンティティを構築するための柔軟性が大幅に向上した。Python SDK v2.28.0 または Node SDK v6.29.0 へアップグレードすることで、この新機能を活用できる。
