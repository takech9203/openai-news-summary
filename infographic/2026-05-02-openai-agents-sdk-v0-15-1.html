技術詳細

WebSocket Keepalive メカニズム

パラメータ 説明 推奨値
keepalive_interval Ping フレーム送信間隔 (秒) 20-30 秒
keepalive_timeout Pong レスポンス待機時間 (秒) 5-10 秒

動作原理: クライアントが定期的に WebSocket Ping フレームを送信し、サーバーから Pong フレームの応答を受け取ることで接続の活性を維持する。keepalive_timeout 内に Pong が返されない場合、接続が切断されたと判断し再接続を試行する。

WebSocket keepalive の使用例
from agents import Agent
from agents.responses_websocket_session import ResponsesWebSocketSession

agent = Agent(
    name="research_agent",
    instructions="You are a research assistant.",
    model="gpt-4o",
)

# keepalive オプションを指定して WebSocket セッションを作成
session = ResponsesWebSocketSession(
    agent=agent,
    keepalive_interval=30,  # 30 秒ごとに Ping 送信
    keepalive_timeout=10,   # Pong の応答待ち 10 秒
)

async with session:
    response = await session.send("Analyze this data...")
    print(response)

PTY SIGINT 修正の技術背景

概念 説明
PTY (疑似端末) Unix でターミナルをエミュレートするデバイスペア。サンドボックス内のプロセス実行に使用
SIGINT 割り込みシグナル (Ctrl+C)。デフォルト動作はプロセス終了
SIG_DFL シグナルのデフォルトハンドラ。子プロセスで明示的にリセットが必要
シグナル継承 fork() で生成された子プロセスは親のシグナルハンドラを継承する
PTY シグナル修正の概念 (unix_local.py)
import signal
import pty

def spawn_sandbox_process(command: str):
    """PTY 子プロセスの生成 (v0.15.1 修正後)"""

    pid, fd = pty.fork()

    if pid == 0:
        # 子プロセス: SIGINT をデフォルトにリセット
        # これにより Ctrl+C で正常終了が可能に
        signal.signal(signal.SIGINT, signal.SIG_DFL)
        os.execvp("/bin/sh", ["/bin/sh", "-c", command])
    else:
        # 親プロセス
        return pid, fd

ユースケース

🔍 長時間リサーチエージェント

複雑な調査タスクを多段階で実行するエージェントセッション。各ステップ間のアイドル時間が発生しても keepalive により WebSocket 接続が維持される。

  • データ収集 → 分析 → レポート生成の多段階ワークフロー
  • ステップ間の処理待ち時間での接続維持
  • プロキシ環境 (企業ネットワーク) での安定動作

💻 ローカルサンドボックス開発

UnixLocal サンドボックスでコードを実行しながら開発する際、Ctrl+C による即座の中断が可能に。

  • 無限ループやハングしたプロセスの正常中断
  • テスト実行中の迅速なキャンセル
  • CI/CD パイプラインでのタイムアウト処理改善

🌐 不安定なネットワーク環境

モバイル環境や公共 Wi-Fi など、接続品質が不安定な環境でのエージェント運用。

  • keepalive による接続死活監視
  • NAT テーブルタイムアウトの回避
  • 早期の切断検知と再接続トリガー

開発者への影響

v0.15.1 は破壊的変更を含まないパッチリリースです。既存のコードは変更なしで動作します。keepalive オプションはデフォルトで従来と同じ動作を維持します。

⬆ アップグレード方法

  • pip install --upgrade openai-agents で v0.15.1 にアップグレード
  • 既存コードの変更は不要 (後方互換性を維持)
  • 必要に応じて keepalive パラメータを追加設定

⚠ 影響を受ける開発者

  • WebSocket セッション利用者: keepalive オプションを活用して接続安定性を向上可能
  • UnixLocal サンドボックス利用者: Ctrl+C の正常動作が自動的に回復
  • Windows ユーザー: クイックスタートドキュメントの改善による恩恵
  • 非英語圏ユーザー: 日本語/韓国語/中国語ドキュメントの更新
