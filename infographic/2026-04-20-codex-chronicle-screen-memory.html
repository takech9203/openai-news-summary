Codex Chronicle: 画面録画から作業コンテキストを自動記憶

2026-04-20

概要

📺

画面録画メモリ

Chronicle はバックグラウンドで macOS の画面録画を実行し、ユーザーの作業画面を継続的に記録する。使用中の IDE、開いているプロジェクト、参照ドキュメントなどの情報を自動的にキャプチャする。

🤖

AI 要約処理

AI エージェントが録画内容を解析し、ユーザーが使用しているツール、取り組んでいるプロジェクト、参照しているドキュメントなどの情報を抽出・要約する。

📝

Markdown ローカル保存

要約はデバイス上に Markdown ファイルとして保存される。Codex が新しいタスクを実行する際、保存された Chronicle メモリを参照してコーディング支援を提供する。

⏱

6 時間後自動削除

録画データは一時的に保持され、6 時間後に自動的に削除される。メモリファイルは保持されるが、元の録画データはプライバシー保護のために削除される。

Before / After

Before

Codex に毎回プロジェクトの背景、使用技術、作業コンテキストを説明する必要があった。複数プロジェクトを並行する場合、コンテキストスイッチのたびに詳細な説明が求められた。

After

Chronicle が画面録画から自動的に作業コンテキストを記憶する。Codex はユーザーの作業状況を理解した上でコーディング支援を提供し、ペアプログラマーに近い文脈理解が可能になる。

主な機能

🎬 Chronicle の仕組み

バックグラウンド録画 → AI エージェントによる要約 → Markdown ファイルとしてローカル保存 → Codex のコンテキストとして活用 → 6 時間後に録画自動削除という 5 段階のフローで動作する。

📋 利用条件

ChatGPT Pro (月額 $200) サブスクライバー向け、macOS のみ対応。オプトインプレビューとして提供され、画面録画権限とアクセシビリティ権限が必要。

🌍 地域制限

EU (GDPR)、英国 (UK GDPR)、スイス (nDSG) では利用不可。画面録画による個人データの大量収集が欧州のデータ保護規制に抵触するため。

🔒 セキュリティ懸念

OpenAI 自身がプロンプトインジェクション攻撃、非暗号化保存、レート制限の急速消費を公式に警告。透明性は評価できるが機能の成熟度への懸念も。

セキュリティ警告

⚠️ OpenAI が公式に警告しているリスク

🚨 プロンプトインジェクション攻撃

Web サイトやドキュメントに埋め込まれた悪意ある命令を Chronicle が記録し、Codex のコンテキストとして取り込む可能性がある。間接的なプロンプトインジェクション攻撃が成立するリスクがある。

🔓 非暗号化のローカル保存

Chronicle のメモリは暗号化されずにデバイスに保存される。端末の盗難やマルウェア感染により、保存されたメモリが第三者にアクセスされるリスクがある。

⚡ レート制限の急速な消費

Chronicle は画面録画の解析に AI 処理を継続的に使用するため、API のレート制限を急速に消費する。ChatGPT Pro の利用枠に大きな影響を与える可能性がある。

開発者への影響

🔄 コーディングワークフローの変革

Chronicle が作業履歴を記憶するため、Codex に対して毎回プロジェクトの背景や使用技術を説明する必要がなくなる。IDE の設定、デバッグ手順、参照ドキュメントなどの暗黙知が自動的に Codex に共有され、ペアプログラマーに近い文脈理解が可能になる。複数プロジェクトを並行する開発者にとって特に大きな生産性向上が見込まれる。

🛡️ セキュリティに関する注意点

画面録画によりプロプライエタリなコード、API キー、認証情報などが Chronicle メモリに記録される可能性がある。信頼できないソースの Web ページを開いた状態での使用は間接的なプロンプトインジェクションのリスクがある。企業環境での採用には情報セキュリティ部門との事前調整が不可欠である。

💰 ChatGPT Pro のコスト対効果

Chronicle による継続的な AI 処理がレート制限を急速に消費するため、ChatGPT Pro ($200/月) のプランにおける実質的なコスト対効果を慎重に評価する必要がある。Chronicle を常時有効にするか、特定の作業セッションのみで有効にするかの運用判断が重要になる。
