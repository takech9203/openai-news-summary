修正内容の詳細

🌐 undici ディスパッチャー不整合ガイダンス

Node.js 組み込み fetch と外部 undici@8 のバージョン不整合を検出し、APIConnectionError メッセージに onRequestStart エラーが原因チェーンに含まれる場合の具体的なガイダンスを追加。元の fetch エラーを cause プロパティとして保持。

💾 text/plain バイナリフォーマット修正

text/plain コンテンツタイプのファイルが format: binary で指定された場合に、テキストではなく raw バイナリデータとして正しく処理されるように修正。CSV、TSV、ログファイルなどに影響。

⚙️ 内部コード生成の更新

自動コード生成 (codegen) に関連する内部的な更新。SDK の型定義やインターフェースの正確性を維持するための継続的なメンテナンス。エンドユーザーへの直接的影響なし。

🧪 回帰テストの追加

undici ディスパッチャー不整合に対する回帰テストを追加し、将来的な品質保証を確立。同様のエッジケースが再発しないことを自動検証。

undici バージョン互換性

環境 undici バージョン ハンドラー API
Node.js 20 組み込み ~5.x 相当 旧ライフサイクル
Node.js 22 組み込み ~6.x 相当 旧ライフサイクル
npm undici@8 8.x 新ライフサイクル (onRequestStart)

Node.js の組み込み globalThis.fetch は内部的に古いバージョンの undici を使用。外部からインストールした undici@8 の ProxyAgent は新しいディスパッチャーハンドラーライフサイクルを期待するため、バージョン不整合が発生する。

正しいプロキシ設定パターン

proxy-setup.js - 推奨構成 
import OpenAI from 'openai';
import { fetch, ProxyAgent } from 'undici';

// undici の fetch と ProxyAgent を同じパッケージから使用する
const client = new OpenAI({
  fetch,  // undici@8 の fetch を明示的に渡す
  fetchOptions: {
    dispatcher: new ProxyAgent('http://proxy.example.com:8080'),
  },
});

// これでバージョン不整合が解消される
const response = await client.chat.completions.create({
  model: 'gpt-4o',
  messages: [{ role: 'user', content: 'Hello!' }],
});
file-upload.js - バイナリアップロード修正 
import OpenAI from 'openai';
import fs from 'fs';

const client = new OpenAI();

// 修正前: text/plain + binary format の場合、
// テキストとして処理されエンコーディング問題が発生

// 修正後: format: binary が指定されていれば
// content-type に関係なく raw バイナリとして転送
const file = await client.files.create({
  file: fs.createReadStream('data.csv'),
  purpose: 'assistants',
});

🔄 破壊的変更なし

まとめ

OpenAI Node SDK v6.39.1 は、プロキシ環境での利用とファイルアップロードに関する 2 つの重要なバグ修正を含むパッチリリースである。undici ディスパッチャーのバージョン不整合問題に対しては、互換性レイヤーの実装ではなく、明確な診断ガイダンスとエラーメッセージの改善というアプローチが採用された。これは「実装固有の fetch オプションを使用する場合は、対応する fetch 実装も提供する」という設計原則を明確化するものであり、SDK の fetch 抽象化レイヤーの設計思想を反映している。text/plain バイナリアップロードの修正と合わせ、エッジケースでの SDK の堅牢性が向上した。プロキシ環境で接続エラーを経験している開発者は速やかなアップグレードを推奨する。
