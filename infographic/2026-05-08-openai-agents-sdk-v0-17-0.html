API UPDATE

OpenAI Agents SDK v0.17.0

2026-05-08

主な変更点

🔊

RealtimeAgent デフォルトモデル変更

RealtimeAgent のデフォルトモデルが gpt-realtime-2 に変更されました。従来のモデルを使用する場合は明示的に指定が必要です。

🔒

Sandbox ローカルソースのセキュリティ強化

LocalFile.src と LocalDir.src が base_dir 内に制限され、外部パスへのアクセスには明示的な SandboxPathGrant が必要になりました。

🐛

バグ修正: context-management extra_args 衝突

Responses API の context-management における extra_args の衝突問題が解消されました。

Before / After

Before

LocalFile.src と LocalDir.src はホスト上の任意のパスに自由にアクセスできた。セキュリティ制限なし。

After

base_dir 内に存在するか、明示的な SandboxPathGrant が付与されている場合のみアクセスが許可される。

主な機能

🎙️ gpt-realtime-2 デフォルト化

RealtimeAgent のデフォルトモデルが最新の gpt-realtime-2 に更新され、音声 AI エージェントのパフォーマンスが向上します。

🛡️ SandboxPathGrant

外部パスへのアクセスを明示的に許可する新しいセキュリティ機構。read_only オプションでさらに制限可能です。

📁 base_dir 制約

Sandbox ワークスペースのファイルアクセスが SDK プロセスの作業ディレクトリ配下に自動的に制限されます。

コードサンプル

Python SandboxPathGrant を使った移行例 
from pathlib import Path

from agents.sandbox import Manifest, SandboxPathGrant
from agents.sandbox.entries import Dir, LocalDir

TRUSTED_DOCS_ROOT = Path("/opt/my-app/docs")

manifest = Manifest(
    extra_path_grants=(
        SandboxPathGrant(path=str(TRUSTED_DOCS_ROOT), read_only=True),
    ),
    entries={
        "fixtures": LocalDir(src=Path("fixtures"), description="Local test fixtures."),
        "docs": LocalDir(src=TRUSTED_DOCS_ROOT, description="Trusted local documents."),
        "output": Dir(description="Generated artifacts."),
    },
)
Python RealtimeAgent の使用例 
from agents import RealtimeAgent

# v0.17.0 以降、デフォルトで gpt-realtime-2 が使用される
agent = RealtimeAgent(name="VoiceAssistant")

# 従来のモデルを明示的に指定する場合
agent_legacy = RealtimeAgent(name="VoiceAssistant", model="gpt-4o-realtime")

開発者への影響

Sandbox を使用しているプロジェクト

base_dir 外のファイルを参照している LocalFile / LocalDir エントリがある場合、SandboxPathGrant を追加する必要があります。未対応の場合、実行時にアクセス拒否エラーが発生します。

RealtimeAgent を使用しているプロジェクト

デフォルトモデルが gpt-realtime-2 に変更されたため、従来の gpt-4o-realtime に依存する動作がある場合はモデルを明示的に指定してください。

context-management で extra_args を使用しているプロジェクト

Responses API の context-management 機能で extra_args を利用していた場合、以前発生していた引数の衝突がこのバージョンで解消されています。特別な対応は不要です。
