from openai import OpenAI

client = OpenAI()

# GPT-5-Codex を使用したコード生成
response = client.chat.completions.create(
    model="gpt-5-codex",
    messages=[
        {
            "role": "system",
            "content": "You are an expert software engineer. Write clean, well-tested code."
        },
        {
            "role": "user",
            "content": "Implement a rate limiter using the token bucket algorithm in Python with async support."
        }
    ],
    temperature=0.2,
    max_tokens=4096
)

print(response.choices[0].message.content)
Python エージェント型タスクでの利用 (ツール呼び出し) 
from openai import OpenAI

client = OpenAI()

# エージェント型コーディングタスクの実行
response = client.chat.completions.create(
    model="gpt-5-codex",
    messages=[
        {
            "role": "system",
            "content": (
                "You are a Codex coding agent. Analyze the repository structure, "
                "identify the issue, implement a fix, and write tests. "
                "Use tool calls to interact with the codebase."
            )
        },
        {
            "role": "user",
            "content": (
                "Fix the race condition in the connection pool manager. "
                "The issue is that concurrent requests can exceed the max pool size."
            )
        }
    ],
    tools=[
        {"type": "function", "function": {"name": "read_file", ...}},
        {"type": "function", "function": {"name": "write_file", ...}},
        {"type": "function", "function": {"name": "run_command", ...}}
    ],
    tool_choice="auto"
)

# エージェントのレスポンスを処理
for choice in response.choices:
    message = choice.message
    if message.tool_calls:
        for tool_call in message.tool_calls:
            print(f"Tool: {tool_call.function.name}")
            print(f"Args: {tool_call.function.arguments}")

ユースケース

1

モデル選択の簡素化

複数バージョンからの選択が不要に。単一のモデル ID (gpt-5-codex) を指定するだけで最適なコーディング支援を受けられる

2

デュアルモードによるワークフロー改善

コード補完などの短時間タスクと大規模リファクタリングなどの長時間タスクを同一モデルで処理。モデル切り替え不要

3

実世界志向の信頼性向上

日常的な開発タスクにおける「使い勝手」が改善。実際の開発シナリオでの信頼性と有用性が向上

4

既存ワークフローからのスムーズな移行

API モデル ID を gpt-5-codex に更新するだけで移行完了。GPT-5.3-Codex サンセットから 4 日後の迅速なリリース

5

Codex プラットフォームとの深い統合

ツール呼び出し、コンテキスト管理、タスク分割など、エージェント型ワークフローの全機能が最適化
