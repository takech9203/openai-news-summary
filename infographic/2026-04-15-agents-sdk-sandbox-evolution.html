Agents SDK の次なる進化: Sandbox Agents によるセキュアなエージェント実行環境

Agents SDK v0.14.0 で導入されたネイティブサンドボックス実行環境とモデルネイティブハーネス -- 永続的かつ隔離されたワークスペースでエージェントをセキュアに実行するための包括的な基盤

📋 概要 (Key Points)

🔒

永続的で隔離されたサンドボックス環境

Sandbox Agents により、エージェントは隔離されたワークスペース上でファイル操作、コマンド実行、リポジトリ編集、アーティファクト生成をセキュアに行えるようになりました。ホスト環境に影響を与えることなく、本番運用が現実的になります。

☁

マルチクラウド対応の実行バックエンド

ローカル (Unix)、コンテナ (Docker)、ホスト型 (Blaxel、Cloudflare、Daytona、E2B、Modal、Runloop、Vercel) の 3 層の実行バックエンドをサポート。開発者は自社インフラに最適なバックエンドを選択できます。

📸

スナップショットとレジュームによる長時間タスク対応

エージェントの実行を中断・再開できるスナップショット/レジューム機能を搭載。大規模なコードレビュー、データ処理、ドキュメント生成など長時間を要するタスクに対応可能です。

🧠

サンドボックスメモリによる知識蓄積

エージェントが過去の実行から学習し、同様のタスクをより効率的に実行できるメモリ機能を搭載。プログレッシブディスクロージャにより、メモリの肥大化を防ぎながら知識を蓄積します。

🔄

既存コードとの互換性

Sandbox Agents は既存の Agent と Runner のフローを拡張する形で実装されており、既存のエージェントコードからの移行コストが低く抑えられています。

🔄 Before / After

Before: 手動の環境セットアップ

  • エージェントがホスト環境に直接アクセス
  • ファイル操作・コマンド実行のセキュリティリスク
  • 長時間タスクの中断・再開が困難
  • 過去の実行から学習する仕組みがない
  • 実行バックエンドが固定的
  • 環境構築のコードが実行ごとに必要

After: Sandbox Agents

  • 隔離されたワークスペースでセキュアに実行
  • Manifest でワークスペースを宣言的に定義
  • スナップショット/レジュームで中断・再開可能
  • サンドボックスメモリで知識を蓄積
  • 7 つのホスト型プロバイダーから選択可能
  • 既存の Agent/Runner パターンを維持

⚙ 主な機能

📦

Sandbox Agents

エージェントに永続的で隔離されたワークスペースを提供する SDK サーフェス。

  • SandboxAgent: デフォルト設定を持つ Agent の拡張
  • Manifest: ファイル、ディレクトリ、Git リポ、環境変数を定義
  • SandboxRunConfig: クライアント作成、レジューム、スナップショット設定
🖥

サンドボックスクライアント

ローカル、コンテナ、ホスト型の 3 層の実行バックエンド。

  • UnixLocalSandboxClient: 高速なローカル開発
  • DockerSandboxClient: コンテナ隔離環境
  • 7 つのホスト型プロバイダー対応
🧠

サンドボックスメモリ

過去の実行から得た教訓を将来の実行に活用するメモリ機能。

  • 読み取り専用/生成専用モードの制御
  • ライブ更新で古いメモリをリアルタイム更新
  • マルチターングルーピング対応
  • S3 バックドメモリで永続化
💾

マウント / スナップショット / レジューム

多様なソースからのファイル提供と実行の中断・再開。

  • ローカル/合成ファイル/Git リポジトリ
  • S3、R2、GCS、Azure Blob Storage
  • ポータブルなスナップショット取得
  • 3 つのレジュームパス
🔍

ランタイム / トレーシング / モデルプラミング

エージェント実行の基盤となるランタイム層の大幅な改善。Runner 管理のサンドボックス準備、サンドボックス対応の RunState シリアライゼーション、統合サンドボックストレーシング、トークン使用量のスパン記録、プロンプトキャッシュキーのデフォルト設定、OpenAI エージェント登録とハーネス ID 設定をサポートします。

💻 コードサンプル

Python 基本的な SandboxAgent の作成と実行 
from agents import SandboxAgent, Runner
from agents.sandbox import Manifest, SandboxRunConfig
from agents.sandbox.clients import DockerSandboxClient

# マニフェストでワークスペースを定義
manifest = Manifest(
    files={"main.py": "print('Hello from sandbox!')"},
    directories=["output"],
    env={"PYTHONPATH": "/workspace"},
)

# SandboxAgent を作成
agent = SandboxAgent(
    name="coding-agent",
    instructions="あなたはコーディングアシスタントです。ファイルを編集し、コマンドを実行してください。",
    default_manifest=manifest,
)

# Docker バックエンドでサンドボックスを実行
sandbox_config = SandboxRunConfig(
    client=DockerSandboxClient(image="python:3.12-slim"),
)

result = await Runner.run(
    agent,
    "main.py を修正して、フィボナッチ数列を計算する関数を追加してください。",
    sandbox_run_config=sandbox_config,
)

print(result.final_output)
Python スナップショットとレジュームによる長時間タスク 
from agents import SandboxAgent, Runner
from agents.sandbox import Manifest, SandboxRunConfig, SandboxSessionState
from agents.sandbox.clients import DockerSandboxClient

agent = SandboxAgent(
    name="long-task-agent",
    instructions="大規模なデータ処理タスクを段階的に実行してください。",
)

sandbox_config = SandboxRunConfig(
    client=DockerSandboxClient(image="python:3.12-slim"),
)

# 初回実行
result = await Runner.run(
    agent,
    "データの前処理を開始してください。",
    sandbox_run_config=sandbox_config,
)

# セッション状態を保存
session_state: SandboxSessionState = result.sandbox_session_state

# 後続の実行でセッションを再開
resume_config = SandboxRunConfig(
    client=DockerSandboxClient(image="python:3.12-slim"),
    session_state=session_state,  # 前回のセッションを復元
)

result = await Runner.run(
    agent,
    "前処理の結果を使って分析を実行してください。",
    sandbox_run_config=resume_config,
)
Python サンドボックスメモリの活用 
from agents import SandboxAgent, Runner
from agents.sandbox import Manifest, SandboxRunConfig
from agents.sandbox.capabilities import SandboxMemory
from agents.sandbox.clients import DockerSandboxClient

# メモリ機能を有効にした SandboxAgent
agent = SandboxAgent(
    name="learning-agent",
    instructions="過去の経験を活かしてタスクを効率的に実行してください。",
    capabilities=[
        SandboxMemory(
            layout="project-alpha",  # メモリを分離するレイアウト名
        ),
    ],
)

sandbox_config = SandboxRunConfig(
    client=DockerSandboxClient(image="python:3.12-slim"),
    group_id="session-001",  # マルチターングルーピング用 ID
)

# 1 回目の実行: エージェントがタスクを学習
result = await Runner.run(
    agent,
    "このプロジェクトのビルドシステムを調査してください。",
    sandbox_run_config=sandbox_config,
)

# 2 回目の実行: 前回の学習を活用
result = await Runner.run(
    agent,
    "テストスイートを実行し、失敗したテストを修正してください。",
    sandbox_run_config=sandbox_config,
)

🎯 ユースケース / 開発者への影響

コード生成・実行エージェントの本番運用

サンドボックス環境により、エージェントがファイルシステム操作やコマンド実行を行ってもホスト環境に影響を与えません。コード生成・実行エージェントの安全な本番運用が現実的になり、開発者はセキュリティリスクを気にせずエージェントを展開できます。

長時間実行タスクと DevOps 自動化

スナップショットとレジューム機能により、大規模なコードレビュー、データ処理、ドキュメント生成など長時間を要するタスクに対応可能。Git リポジトリのマウント、Docker コンテナによる実行、リモートストレージとの連携により、CI/CD パイプラインや DevOps ワークフローにエージェントを容易に組み込めます。

マルチクラウド展開とコスト最適化

7 つのホスト型プロバイダーと 4 つのリモートストレージサービスにより、開発者は自社のインフラストラクチャに最適なバックエンドを選択できます。ローカル開発から本番環境まで同一のコードで動作し、トークン使用量のスパン記録とプロンプトキャッシュキーのデフォルト設定でコスト管理も改善されています。

エージェントの学習と知識蓄積

サンドボックスメモリ機能により、エージェントは過去の実行から学習し、同様のタスクをより効率的に実行できるようになります。プログレッシブディスクロージャにより必要に応じて詳細な情報を段階的に提供し、メモリの肥大化を防ぎます。

📝 まとめ

Agents SDK v0.14.0 で導入された Sandbox Agents は、エージェント開発における安全性、永続性、スケーラビリティの課題を包括的に解決する大規模なアップデートです。SandboxAgent、Manifest、SandboxRunConfig を中心とするアーキテクチャにより、開発者は既存の Agent/Runner パターンを維持しながら、隔離されたワークスペースでのファイル操作、コマンド実行、リポジトリ編集を安全に行えるようになりました。この進化により、Agents SDK はコード生成、データ処理、DevOps 自動化、ドキュメント生成など、セキュアで長時間実行可能なエージェントを構築するための本格的なプラットフォームへと成長しました。
