Braintrust が Codex で顧客リクエストをコードに変換

2026-05-29

概要

🚀

Codex + GPT-5.5 ワークフロー

Braintrust は OpenAI の Codex エンジンと GPT-5.5 を組み合わせ、顧客からの機能リクエストを自動的にコードへ変換するワークフローを構築しました。

🔄

顧客リクエスト → コードパイプライン

顧客からのフィードバックや機能リクエストが Issue として登録されると、Codex が自動的にトリアージし、実装コードとテストコードを生成して Pull Request を作成します。

🧪

Eval プラットフォームでのドッグフーディング

Braintrust は自社の Eval (評価) プラットフォームを使って Codex が生成したコードの品質を自動検証。AI で AI を評価するドッグフーディング手法を実践しています。

⚡

実験の加速

Codex の並列処理能力により、複数の実装アプローチを同時に試すことが可能になり、開発実験のサイクルが大幅に短縮されました。

Before / After

Before

数日
  • 顧客リクエストを手動で Issue 化
  • エンジニアが優先度を判断
  • 設計・実装に数日を要する
  • テスト作成も手動
  • レビュー待ちでボトルネック

After (Codex 導入後)

数時間
  • Codex が自動で Issue トリアージ
  • GPT-5.5 が実装プランを策定
  • サンドボックスで自動コード生成
  • テストコードも同時に生成
  • Eval で品質を即座に検証

主な機能

🔍 Issue トリアージ

顧客からの機能リクエストを自動的に分類・優先度付けし、実装可能なタスクに分解します。

📋 実装プランニング

GPT-5.5 がコードベースを分析し、最適な実装方針とアーキテクチャ変更を提案します。

💻 コード生成

Codex エンジンがサンドボックス環境で実装コードを生成。既存コードとの整合性を保ちます。

✅ 自動評価

Braintrust の Eval フレームワークで生成コードの品質・正確性・パフォーマンスを自動評価します。

📦 PR 作成

実装コード、テストコード、変更説明を含む Pull Request を自動作成し、レビュー待ち状態にします。

コードサンプル

Python Braintrust Eval + Codex 統合例 
import braintrust
from openai import OpenAI

client = OpenAI()

# Codex で生成されたコードを Braintrust Eval で評価
@braintrust.traced
def evaluate_codex_output(task_description: str, generated_code: str):
    """Codex が生成したコードの品質を自動評価"""
    eval_result = client.chat.completions.create(
        model="gpt-5.5",
        messages=[
            {
                "role": "system",
                "content": "コードの正確性、可読性、テストカバレッジを評価してください。"
            },
            {
                "role": "user",
                "content": f"タスク: {task_description}\n\nコード:\n{generated_code}"
            }
        ]
    )
    return eval_result.choices[0].message.content

# Braintrust Eval フレームワークでバッチ評価
experiment = braintrust.init(project="codex-quality")

for task in customer_requests:
    with experiment.start_span(name="codex-eval") as span:
        # Codex でコード生成
        code = codex_generate(task["description"])
        # 生成コードを評価
        score = evaluate_codex_output(task["description"], code)
        span.log(
            input=task["description"],
            output=code,
            scores={"quality": score}
        )

print(experiment.summarize())

ユースケース

💬 顧客フィードバック即座実装

顧客からの機能リクエストを受け取ると、Codex が数時間以内に実装コードを生成。従来数日かかっていた初期実装が劇的に短縮され、顧客満足度が向上します。

🤖 AI 評価自動化

Braintrust の Eval プラットフォームと連携し、Codex が生成したコードの品質を自動的に評価。人間のレビュー前に品質ゲートを設け、レビュー負荷を軽減します。

🏭 バッチ処理並列開発

複数の顧客リクエストを Codex のバッチ処理で同時に実装。並列開発により、スプリント内で処理できるタスク数が大幅に増加しました。
