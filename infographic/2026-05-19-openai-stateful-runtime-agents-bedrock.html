OpenAI が Amazon Bedrock 向けステートフルランタイム環境を発表

2026-05-19

概要

🧠

Stateful Runtime Environment

Amazon Bedrock 上で AI エージェントが永続的なメモリと状態を保持できる新しいランタイム環境を提供

💾

永続メモリ / 状態管理

長時間実行されるエージェントセッションで、コンテキストと状態を自動的に保持。中断からの復旧にも対応

☁️

ネイティブ AWS 統合

IAM、VPC、CloudWatch、Lambda とシームレスに連携し、エンタープライズグレードのセキュリティと監視を実現

🌐

マルチクラウド戦略

Azure + AWS + Dell オンプレミスをカバーするマルチクラウド展開の一環として発表

🔧

OpenAI Agents SDK 対応

OpenAI Agents SDK のパターンが AWS 上でネイティブに動作し、既存のエージェント設計を容易に移植可能

Before / After

Before

  • ステートレスな API コール
  • リクエストごとにコンテキストが消失
  • 長時間タスクの中断で再開不可
  • ツール呼び出し履歴が残らない
  • 手動でのセッション管理が必要

After

  • ステートフルエージェントが永続メモリを保持
  • セッション間でコンテキスト自動継続
  • 中断タスクを自動的に復旧・再開
  • 全てのツール呼び出し履歴を記録
  • DynamoDB ベースの自動状態管理

ランタイム機能

💾 永続メモリ

セッション間でメモリを自動保持。過去のやりとりや分析結果を参照して継続的なタスクを実行

📋 状態管理

マルチステップタスクの進捗状態を DynamoDB に保存。複雑なワークフローを確実に実行

🔄 セッション復旧

中断されたタスクを自動検知し、最後の状態から再開。ネットワーク障害やタイムアウトに強い

📝 ツール呼び出し履歴

全てのツール実行履歴を記録し、デバッグや監査に活用。実行のトレーサビリティを確保

📈 オートスケーリング

AWS Auto Scaling により、負荷に応じてリソースを自動調整。コスト効率とパフォーマンスを最適化

🔒 エンタープライズセキュリティ

IAM によるアクセス制御と VPC 内での実行により、企業レベルのセキュリティ要件に対応

コードサンプル

Python AWS Bedrock ステートフルエージェント実装例 
import boto3
from openai import OpenAI

# AWS Bedrock 経由で OpenAI エージェントを利用
bedrock = boto3.client("bedrock-agent-runtime")

# ステートフルセッションの作成
session = bedrock.create_agent_session(
    agentId="openai-agent-001",
    sessionAttributes={
        "runtime": "openai-stateful",
        "model": "gpt-4o",
        "persistState": True
    }
)

# エージェントの実行 (状態が自動的に保持される)
response = bedrock.invoke_agent(
    agentId="openai-agent-001",
    sessionId=session["sessionId"],
    inputText="前回の分析結果を踏まえて、次のステップを提案してください"
)

ユースケース

AWS エンタープライズ環境

既存の AWS インフラ上でセキュアに AI エージェントを運用。IAM ポリシー、VPC、CloudWatch による統合管理で、コンプライアンス要件を満たしながら高度な自律エージェントを展開

マルチステップワークフロー

データ分析、レポート生成、承認フローなど複数ステップにわたるタスクを、状態を保持しながら確実に実行。中断時も最後のチェックポイントから自動再開

マルチクラウド AI 戦略

Azure、AWS、Dell オンプレミスにまたがる統一的な AI エージェント基盤を構築。OpenAI Agents SDK の共通パターンにより、環境を問わず同一のエージェント設計を適用可能
