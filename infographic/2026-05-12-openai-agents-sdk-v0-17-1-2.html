コードサンプル

Python Realtime ツール承認のスコープ分離 (v0.17.1 修正) 
# Realtime ツール承認のスコープ分離 (v0.17.1 修正)
from agents import RealtimeAgent

agent_a = RealtimeAgent(
    name="AssistantA",
    tools=["web_search", "code_exec"],
    tool_approval_required=True,
)

agent_b = RealtimeAgent(
    name="AssistantB",
    tools=["file_read"],
    tool_approval_required=True,
)
# 各エージェントのツール承認が独立して管理される
Python AudioInput バッファ安全性 (v0.17.1 修正) 
# AudioInput バッファ安全性 (v0.17.1 修正)
from agents.voice import AudioInput

audio_buffer = bytearray(b"\x00\x01\x02\x03...")
base64_data = audio_input.to_base64()
# v0.17.1以降: 元のバッファは変更されない
assert audio_buffer == bytearray(b"\x00\x01\x02\x03...")

Before / After

Before (v0.17.0)

  • AudioInput.to_base64() が元のバッファを破壊的に変更
  • セッションに破損データが含まれるとクラッシュ
  • トレーシングのシャットダウン時にデータ消失の可能性
  • 複数エージェントのツール承認が共有状態で競合
  • handoff 時に会話履歴が欠落

After (v0.17.2)

  • バッファのコピーを作成し元データを安全に保持
  • 破損アイテムを graceful にスキップして継続
  • ベストエフォート・シャットダウンで確実にフラッシュ
  • 各エージェント固有のスコープで承認を管理
  • handoff 時の会話履歴を完全に保持

アップグレードが必要なユーザー

🎤 Realtime Agents ユーザー

音声入力のバッファ安全性とツール承認のスコープ分離が修正されました。複数の RealtimeAgent を並行利用している場合は即時アップグレードを推奨します。

💾 MongoDB / Redis セッションユーザー

タイムスタンプ保持の問題と破損データ時のクラッシュが修正されました。本番環境でセッション永続化を利用している場合は v0.17.2 への更新が必須です。

🤝 マルチエージェント handoff ユーザー

エージェント間の会話履歴保持と guardrail のクリーンアップが修正されました。複数エージェントを連携させるワークフローの信頼性が大幅に向上します。
