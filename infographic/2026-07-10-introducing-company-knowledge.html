新機能 / Enterprise

Introducing Company Knowledge

ChatGPT Enterprise / Team 向けナレッジベース統合機能

2026-07-10 | OpenAI News/Blog

★ 概要

📚
組織知の統合: 社内ドキュメント、ポリシー、プロセス文書を ChatGPT に接続し、社内情報に基づいた正確な回答を生成する RAG ベースのエンタープライズソリューション
🔍
自然言語検索: 従来は社内 Wiki や文書管理システムに散在していた組織知を、自然言語で検索・活用できる環境を実現
🔒
エンタープライズグレードのセキュリティ: ソースシステム権限の継承、RBAC、監査ログ、データ分類による厳密なアクセス制御
⚡
GPT-5.6 Sol 活用: 1.05M トークンコンテキストによる大規模ドキュメントの横断的分析と高精度な回答生成が可能
🔗
豊富なコネクタ: Confluence、SharePoint、Google Drive、Notion など 8 以上の主要エンタープライズツールとネイティブ統合

⇄ 導入前後の変化

BEFORE
  • 社内 Wiki に情報が散在
  • 手動検索で時間を浪費
  • 新入社員が情報を見つけられない
  • 部門間の知識共有が困難
  • サードパーティ RAG 構築が必要
  • 権限管理が複雑で断片化
AFTER
  • 統一ナレッジベースに集約
  • AI が即座に正確な回答を生成
  • 自然言語で自律的に情報取得
  • 組織横断の知識統合が実現
  • マネージド RAG サービスで運用
  • ソース権限の自動継承で簡素化

  • Custom GPTs との統合

💻 技術詳細 - API サンプルコード

Company Knowledge を利用した Chat Completions

from openai import OpenAI

client = OpenAI()

# Company Knowledge を参照した回答生成
response = client.chat.completions.create(
    model="gpt-5.6-sol",
    messages=[
        {
            "role": "system",
            "content": "You are a helpful assistant that answers questions using "
                       "the company's internal knowledge base. Always cite sources."
        },
        {
            "role": "user",
            "content": "What is our company's policy on remote work? "
                       "Include details about equipment allowance."
        }
    ],
    tools=[
        {
            "type": "company_knowledge",
            "company_knowledge": {
                "knowledge_base_ids": ["kb_abc123", "kb_def456"],
                "max_results": 10,
                "score_threshold": 0.7,
                "include_metadata": True
            }
        }
    ]
)

# 回答とソース引用の取得
message = response.choices[0].message
print(message.content)

# ソース情報の確認
if message.annotations:
    print("\n--- Sources ---")
    for annotation in message.annotations:
        print(f"  [{annotation.index}] {annotation.source.title}")
        print(f"      URL: {annotation.source.url}")
        print(f"      Relevance: {annotation.source.score:.2f}")

🛠 開発者への影響

  • RAG 基盤のファーストパーティ化: サードパーティツールで構築していた RAG パイプラインが OpenAI のマネージドサービスとして提供。自前の embedding、チャンキング、検索インフラの構築・運用が不要に
  • Enterprise API の拡張: Chat Completions API に company_knowledge ツールタイプが追加され、既存アプリケーションへの統合が容易に
  • 権限管理の簡素化: ソースシステムの権限を自動継承する機能により、独自のアクセス制御レイヤーの構築が不要に
  • Custom GPTs エコシステムへの影響: 部門特化型 AI アシスタントの構築が格段に容易に。社内ヘルプデスク、オンボーディングアシスタントなどの実装ハードルが低下
  • Codex / Workspace Agents 連携: Codex が社内コーディング規約を参照し、組織の開発標準に準拠したコード生成を実現
  • データガバナンスの強化: 監査ログ、データ分類、保持ポリシーにより、規制の厳しい業界での採用障壁が低下
  • コスト構造の変化: ストレージ容量、インデックスサイズ、検索クエリ数に基づく従量課金。自前構築との比較検討が必要

💡 ユースケース

社内ヘルプデスク / IT サポート
社内の IT ポリシー、トラブルシューティングガイド、FAQ を Company Knowledge に登録することで、社員は ChatGPT に自然言語で問い合わせ、即座に正確な回答を取得。ヘルプデスクチームの問い合わせ対応負荷を大幅に削減。
新入社員オンボーディング
就業規則、福利厚生、社内システムの利用方法、部門固有のプロセスなどを集約。新入社員が自律的に必要な情報を取得でき、メンターやマネージャーの説明コストを削減しつつ、オンボーディング体験を向上。
技術ドキュメント検索
API 仕様書、アーキテクチャ決定記録 (ADR)、コーディング規約、デプロイメント手順書などを統合。エンジニアが開発中に即座に参照でき、Codex との連携により社内標準に準拠したコード生成も実現。
コンプライアンス / 法務支援
規制文書、内部統制ポリシー、契約テンプレート、過去の法務判断を集約し、法務チームの調査効率を向上。回答にはソース引用が付与されるため、根拠の確認が容易。

✓ まとめ

  1. マネージド RAG サービス: ドキュメントのアップロード、チャンキング、エンベディング、検索、回答生成までをエンドツーエンドで管理するファーストパーティソリューション
  2. 豊富なコネクタ: Confluence、SharePoint、Google Drive、Notion など主要なエンタープライズツールとのネイティブ統合
  3. エンタープライズグレードのセキュリティ: ソースシステム権限の継承、RBAC、監査ログ、データ分類による厳密なアクセス制御
  4. 既存機能との連携: Custom GPTs、Workspace Agents、Codex との統合により、組織の知識を活用した AI ワークフローを包括的に実現
  5. GPT-5.6 Sol の能力活用: 1.05M トークンコンテキストによる大規模ドキュメントの横断的分析と、高精度な回答生成

本機能は、ChatGPT Enterprise のバリュープロポジションを「汎用 AI アシスタント」から「組織の知識を体現する AI パートナー」へと進化させるものであり、エンタープライズ市場における OpenAI の競争優位性を強化する戦略的なリリースである。
