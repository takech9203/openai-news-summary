API エンドポイント

Endpoint Bedrock Responses API 
https://bedrock-mantle.{region}.api.aws/openai/v1/responses

BedrockOpenAI クライアントの基本利用

Python Codex モデルを使用したコード生成 
from openai import BedrockOpenAI

# BedrockOpenAI クライアントの初期化
client = BedrockOpenAI(aws_region="us-east-2")

# Codex モデルを使用したコード生成
response = client.responses.create(
    model="openai.codex",
    input="Python で REST API のレート制限ミドルウェアを実装してください。",
)

print(response.output_text)

トークンプロバイダー方式 (本番推奨)

Python 動的トークン取得による認証 
from aws_bedrock_token_generator import provide_token
from openai import BedrockOpenAI

# AWS 認証情報チェーンを利用した動的トークン取得
client = BedrockOpenAI(
    aws_region="us-east-2",
    bedrock_token_provider=provide_token,
)

# GPT-5.5 を使用した本番アプリケーション
response = client.responses.create(
    model="openai.gpt-5.5",
    input="顧客サポートチケットを分類し、優先度を判定してください。",
)

print(response.output_text)

構造化出力 (GPT-5.4)

Python Pydantic + JSON Schema による型安全な出力 
from openai import BedrockOpenAI
from pydantic import BaseModel

client = BedrockOpenAI(aws_region="us-east-2")


class TaskClassification(BaseModel):
    category: str
    priority: str
    estimated_hours: float
    assignee_team: str


response = client.responses.create(
    model="openai.gpt-5.4",
    input="新規顧客のオンボーディングフロー改善: レスポンス時間を50%短縮する",
    text={
        "format": {
            "type": "json_schema",
            "name": "task_classification",
            "schema": TaskClassification.model_json_schema(),
        }
    },
)

print(response.output_text)

ユースケース

🤖 AI エージェントの本番展開

Codex モデルを活用し、コードレビュー、脆弱性検出、PR 自動生成などのソフトウェアエンジニアリングエージェントを AWS 環境内で本番稼働させることが可能。既存の IAM ポリシーとセキュリティ制御をそのまま適用できる。

🏢 エンタープライズ業務自動化

GPT-5.5 を利用した顧客サポートチケットの分類・優先度判定、GPT-5.4 による構造化データ抽出など、企業の既存 AWS ワークフロー内でAI を活用した業務自動化を即座に展開可能。

🔄 既存システムからのシームレス移行

初期リリースで評価を行っていた開発者は、コードの変更なしにそのまま本番利用に移行可能。BedrockOpenAI クライアントのインターフェースに変更はなく、AWS Marketplace 経由の調達プロセスも利用できる。
