OpenAI Python SDK v2.35.0/v2.35.1: Image 2 API の更新とレガシー CLI の削除

gpt-image-2 モデル対応のパラメータ更新、レガシー Python CLI の完全削除 (破壊的変更)、top_logprobs パラメータの説明更新、および size enum リグレッション修正のホットフィックスを含むリリース

⚠️

破壊的変更: レガシー CLI の完全削除

v2.35.0 で openai パッケージに同梱されていたレガシー Python CLI が完全に削除されました。これまで openai api コマンドを使用していたワークフローは動作しなくなります。

  • 影響を受けるコマンド: openai api completions.create, openai api chat_completions.create など
  • 代替手段: Python スクリプトまたは curl コマンドを使用して API を直接呼び出してください
  • CI/CD パイプラインでレガシー CLI を使用している場合は、スクリプトへの移行が必要です

📋 概要 (Key Points)

🖼

Image Generation API (gpt-image-2) パラメータ更新

gpt-image-2 モデルに対応した画像生成 API のパラメータが更新されました。新しい size オプションや quality パラメータの型定義が改善され、より正確な型安全性が提供されます。

🚫

レガシー Python CLI の完全削除 (破壊的変更)

長らく非推奨だったレガシー CLI が完全に削除されました。openai api コマンドに依存するスクリプトやワークフローは直ちに移行が必要です。

📊

top_logprobs パラメータの説明更新

Chat Completions API の top_logprobs パラメータに関するドキュメンテーションと型定義の説明が更新され、開発者がパラメータの用途をより正確に理解できるようになりました。

🔧

v2.35.1 ホットフィックス: size enum リグレッション修正

v2.35.0 で導入された size enum の型定義にリグレッションが発見され、v2.35.1 で即座に修正されました。画像生成時の size パラメータが正しく動作するようになっています。

💻 コードサンプル

Python gpt-image-2 で画像を生成 
from openai import OpenAI

client = OpenAI()

# gpt-image-2 モデルで画像を生成
response = client.images.generate(
    model="gpt-image-2",
    prompt="A serene Japanese garden with cherry blossoms",
    size="1024x1024",  # v2.35.1 で修正された size enum
    quality="hd",
    n=1,
)

image_url = response.data[0].url
print(f"Generated image URL: {image_url}")

🛠 開発者への影響

画像生成 API 利用者

  • gpt-image-2 の活用: 新しいモデルに対応した型定義により、IDE の自動補完やバリデーションがより正確に機能し、開発体験が向上する
  • size enum の修正: v2.35.1 で修正された size パラメータにより、"1024x1024" などの指定が型エラーなく動作する
  • quality パラメータ: HD 品質の画像生成を型安全に指定可能

レガシー CLI 利用者 (即時対応が必要)

  • CI/CD パイプライン: openai api コマンドを使用しているビルドスクリプトは即座に動作しなくなるため、Python スクリプトまたは curl への移行が必須
  • 開発ワークフロー: ローカル開発で CLI を使用していた場合、SDK の Python API を直接呼び出す方法に切り替える必要がある
  • ドキュメント更新: 社内ドキュメントやチュートリアルでレガシー CLI を参照している箇所の更新が必要

Chat Completions API 利用者

  • top_logprobs の明確化: パラメータの説明が改善され、各トークンに対して返される上位 N 個のログ確率の用途がより明確になった
  • 型定義の精度向上: パラメータの有効範囲や組み合わせが型レベルで表現され、実行時エラーを事前に防止できる
