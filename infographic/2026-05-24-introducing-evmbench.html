OpenAI と Paradigm がスマートコントラクト脆弱性検出 AI ベンチマーク「EVMbench」を発表

2026-05-24

概要

🤝

OpenAI と Paradigm の共同発表

AI 研究のリーダーと暗号資産投資ファームが協力し、スマートコントラクトセキュリティの新たな評価基準を策定

🛡️

AI エージェントのセキュリティ能力を評価

スマートコントラクトの脆弱性を検出、修正、悪用する AI エージェントの能力を体系的に測定するベンチマーク

📋

3 つの評価タスク

脆弱性検出 (Detection)、脆弱性修正 (Patching)、エクスプロイト生成 (Exploitation) の 3 軸で評価

⚠️

高重大度の脆弱性に特化

High Severity のスマートコントラクト脆弱性を対象とし、実際の DeFi セキュリティに直結する評価を実現

🌐

DeFi セキュリティの向上に貢献

分散型金融エコシステム全体のセキュリティ水準を引き上げる標準化されたベンチマークとして機能

評価タスク

🔍 脆弱性検出 (Detection)

コントラクトコードから重大な脆弱性を発見する能力を評価。AI エージェントがソースコードを解析し、潜在的なセキュリティリスクを特定できるかを測定します。

🔧 脆弱性修正 (Patching)

発見された脆弱性に対して適切なパッチを作成する能力を評価。セキュリティを確保しつつ、コントラクトの機能を維持する修正が行えるかを測定します。

💥 エクスプロイト生成 (Exploitation)

脆弱性を実際に悪用するコードを生成する能力を評価。攻撃者の視点を理解し、実行可能なエクスプロイトを構築できるかを測定します。

対象脆弱性タイプ

EVMbench が評価対象とする主要なスマートコントラクト脆弱性カテゴリ

Solidity EVMbench 評価対象の脆弱性例 (再入攻撃) 
// EVMbench 評価対象の脆弱性例 (再入攻撃)
contract VulnerableVault {
    mapping(address => uint256) public balances;

    function withdraw(uint256 amount) external {
        require(balances[msg.sender] >= amount, "Insufficient balance");
        // 脆弱性: 状態更新前の外部呼び出し
        (bool success, ) = msg.sender.call{value: amount}("");
        require(success, "Transfer failed");
        balances[msg.sender] -= amount;
    }
}

ユースケース

DeFi プロトコルのデプロイ前自動セキュリティ監査

プロトコルの本番デプロイ前に AI エージェントが自動的にセキュリティ監査を実施し、重大な脆弱性を事前に検出。開発サイクルの早い段階で問題を発見することで、被害を未然に防止します。

既存プロトコルの継続的脆弱性モニタリング

デプロイ済みのプロトコルに対して継続的に脆弱性スキャンを実行し、新たに発見された攻撃パターンやロジックエラーをリアルタイムで検出。既存資産の保護を強化します。

セキュリティ研究者の初期スクリーニング自動化

監査対象のコントラクトに対する初期スクリーニングを AI が自動化し、研究者がより深い分析に集中できる環境を構築。人間の専門知識と AI の網羅性を組み合わせた効率的なワークフローを実現します。
