4o Image Generation の導入

2026-06-18

主要ポイント

🎨

GPT-4o ネイティブ画像生成

画像生成が GPT-4o に直接統合され、テキスト理解と画像生成がシームレスに連携

⚡

ストリーミング対応

partial_images パラメータにより、生成途中の画像を 0-3 枚リアルタイムで取得可能

🔄

マルチターン編集

Responses API で会話コンテキストを維持し、既存画像を自然言語で繰り返し編集

🔌

2 つの API エンドポイント

Images API (/v1/images/generations) と Responses API (/v1/responses) の両方で利用可能

💰

トークンベース課金

サイズと品質に応じた柔軟なトークンベースの料金体系

画像生成モデルの進化

主な機能

🧠 ネイティブ統合

GPT-4o の言語理解能力と画像生成が一体化。プロンプトの意図をより正確に反映した画像を生成

📡 ストリーミング

partial_images パラメータ (0-3) で生成途中の中間画像をリアルタイム配信。UX の大幅改善

💬 マルチターン編集

Responses API の会話コンテキストを活用し、「もっと明るく」「背景を変更」など自然言語で繰り返し編集

📐 豊富な出力形式

PNG / WebP / JPEG フォーマット、透明背景対応、複数サイズから選択可能

コードサンプル

Python Images API - 基本的な画像生成 
from openai import OpenAI
client = OpenAI()

response = client.images.generate(
    model="gpt-4o",
    prompt="A sunset over Tokyo skyline",
    size="1536x1024",
    quality="high",
    output_format="png",
)
Python Responses API - ツール経由の画像生成 
response = client.responses.create(
    model="gpt-4o",
    input="Generate an image of a cat on a windowsill",
    tools=[{"type": "image_generation"}],
)
Python ストリーミング + partial_images 
from openai import OpenAI
client = OpenAI()

# ストリーミングで中間画像を取得
stream = client.responses.create(
    model="gpt-4o",
    input="A watercolor painting of cherry blossoms",
    tools=[{"type": "image_generation"}],
    stream=True,
    partial_images=3,  # 生成中に最大 3 枚の中間画像を配信
)

for event in stream:
    if event.type == "response.image.partial":
        print(f"中間画像を受信: {len(event.data)} bytes")
    elif event.type == "response.image.done":
        print("最終画像が完成しました")

ユースケース

🎨 クリエイティブツール

デザイナーやクリエイターがアイデアを即座に視覚化。マルチターン編集により、細部の調整を会話形式で繰り返し実施可能。コンセプトアートの探索やムードボード作成に最適

🛒 E コマース

商品画像の背景変更、バリエーション生成、プロモーション素材の大量作成。透明背景オプションにより、既存のカタログシステムとの統合が容易

💬 マルチターン編集ワークフロー

Responses API の会話コンテキストを活用し、「色を変更」「テキストを追加」「スタイルを調整」など段階的な画像改善。非デザイナーでも高品質な画像を制作可能
