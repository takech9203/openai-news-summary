API Update

OpenAI SDK アップデート: Computer Action に keys フィールド追加

2026-03-25

概要

🎯

Computer Action に keys フィールド追加

Click、DoubleClick、Drag、Move、Scroll の各アクションで修飾キー (Ctrl、Shift、Alt) を指定可能に。修飾キー + マウス操作を単一アクションとして実行できるようになった。

🔄

WebSocket 非同期イテレータ対応 (Node SDK)

Node SDK の WebSocket クラスに async iterator と stream() メソッドを追加。Realtime API のストリーミングイベントを for await...of 構文で処理可能に。

🔧

レスポンス型の修正と厳格化

SDK のレスポンス型を拡張されたアイテムスキーマに合わせて修正。ResponseInputMessageItem の type フィールドが必須化された。

📦

Python v2.30.0 / Node v6.33.0 同時リリース

Python SDK と Node SDK が同日にリリースされ、両 SDK で Computer Action の keys フィールドとレスポンス型の修正が同期的に提供された。

Before / After 比較

❌ Before

修飾キーとマウス操作を個別のアクションとして実行する必要があった。

  • key_down("ctrl") を送信
  • click(x, y) を送信
  • key_up("ctrl") を送信
  • タイミングずれのリスクあり
  • アクション数が増加し冗長

✅ After

keys フィールドで修飾キー付きマウス操作を単一アクションとして実行可能に。

  • click(x, y, keys=["ctrl"]) で完結
  • タイミングずれのリスク解消
  • コードがシンプルに
  • Ctrl+Click、Shift+Drag 等を直接指定
  • 全 5 アクションタイプで対応

主な機能

⌨️ keys フィールド

Click、DoubleClick、Drag、Move、Scroll で Ctrl、Shift、Alt 等の修飾キーを指定可能。

  • 複数キーの同時指定対応
  • Ctrl+Shift+Click も可能
  • 全プラットフォーム対応

🔌 WebSocket stream()

for await...of 構文でストリーミングイベントを処理。Node SDK の Realtime API 利用が大幅に簡素化。

  • async iterator パターン
  • イベントループとの統合
  • エラーハンドリング改善

📐 型定義の厳格化

ResponseInputMessageItem の type フィールドが必須化。型安全性が向上し、実行時エラーの早期検出が可能に。

  • type フィールド必須化
  • 拡張アイテムスキーマ対応
  • IDE 補完の改善

🧪 テスト基盤の刷新

モック API サーバーを Prism から Steady に移行。テストの安定性と速度が向上。

  • Prism から Steady へ移行
  • テスト実行速度の改善
  • CI パイプラインの安定化

コードサンプル

Python Computer Use with modifier keys (v2.30.0) 
from openai import OpenAI

client = OpenAI()

# Computer Use with modifier keys (new keys field)
response = client.responses.create(
    model="computer-use-sonnet",
    tools=[{
        "type": "computer_use_preview",
        "display_width": 1920,
        "display_height": 1080,
        "environment": "browser"
    }],
    input=[...]
)

# Actions now support keys:
# {"type": "click", "x": 500, "y": 300, "keys": ["ctrl"]}
# {"type": "drag", "start_x": 100, ..., "keys": ["shift"]}
TypeScript WebSocket async iterator (v6.33.0) 
import OpenAI from "openai";

const client = new OpenAI();

// Async iterator with WebSocket (new in v6.33.0)
const ws = await client.realtime.connect({ model: "gpt-5.4-realtime" });

for await (const event of ws.stream()) {
    if (event.type === "response.text.delta") {
        process.stdout.write(event.delta);
    }
}

開発者への影響

🤖 Computer Use エージェントの高度化

keys フィールドの追加により、Computer Use エージェントがより複雑なデスクトップ操作を実行可能に。

  • Ctrl+Click によるリンクの新しいタブでの展開
  • Shift+Drag によるファイルの複数選択
  • Alt+Click によるコンテキストメニュー操作
  • 修飾キーの組み合わせによる生産性向上

⚡ リアルタイム通信の改善

Node SDK の WebSocket async iterator により、Realtime API のストリーミング処理がより直感的に記述可能。

  • for await...of による簡潔なイベント処理
  • コールバック地獄の回避
  • エラーハンドリングの統一
  • 既存の async/await パターンとの統合

⚠️ SDK アップグレード時の注意点

型定義の厳格化に伴い、既存コードの修正が必要になる場合がある。

  • ResponseInputMessageItem に type フィールドの明示的な指定が必須
  • 型チェックが厳格化されたためコンパイルエラーの可能性
  • テスト基盤が Prism から Steady に変更 (コントリビューター向け)
  • 互換性テストの実施を推奨
