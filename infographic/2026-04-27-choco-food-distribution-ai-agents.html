Before / After

Before

  • 手動での注文入力・処理
  • 営業時間内のみ注文受付
  • 顧客固有の暗黙知に依存
  • 人員増加でしかスケールできない

主要プロダクト

📋 OrderAgent

マルチモーダル入力 (メール、SMS、画像、ドキュメント) を ERP 対応の構造化注文データに変換

  • 動的インコンテキスト学習
  • 顧客固有の SKU 解決
  • 設定可能な自動化閾値

🎙️ VoiceAgent

OpenAI Realtime API による電話注文システム。サブ秒レイテンシで営業時間外の注文受付を実現

  • リアルタイム音声処理
  • 自然な対話体験
  • 24 時間 365 日稼働

Python OrderAgent: マルチモーダル注文処理の概念例 
from openai import OpenAI

client = OpenAI()

# OrderAgent: マルチモーダル注文処理
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {
            "role": "system",
            "content": (
                "You are an order processing agent for food distribution. "
                "Parse incoming orders from various formats and output "
                "structured ERP-ready order data. Use customer-specific "
                "SKU mappings and unit preferences."
            )
        },
        {
            "role": "user",
            "content": [
                {"type": "text", "text": "Process this order from customer #1234:"},
                {
                    "type": "image_url",
                    "image_url": {"url": "https://example.com/order-image.jpg"}
                }
            ]
        }
    ],
    tools=[
        {
            "type": "function",
            "function": {
                "name": "create_erp_order",
                "description": "Create a structured order in the ERP system",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "customer_id": {"type": "string"},
                        "items": {
                            "type": "array",
                            "items": {
                                "type": "object",
                                "properties": {
                                    "sku": {"type": "string"},
                                    "quantity": {"type": "number"},
                                    "unit": {"type": "string"}
                                }
                            }
                        },
                        "delivery_date": {"type": "string"}
                    },
                    "required": ["customer_id", "items"]
                }
            }
        }
    ]
)

リーダーシップからの教訓

1

初日から評価を始める

10-20 件のグラウンドトゥルースデータからでも、評価フレームワークを構築し精度を定量的に測定することが重要

2

AI ネイティブなオブザーバビリティに投資する

モデルの入力、出力、推論トレースを可視化する監視基盤を構築し、本番運用の品質を担保する

3

早期に適切な期待値を設定する

LLM は確率的であり決定論的ではないことをステークホルダーに伝え、設定可能な自動化閾値で柔軟に運用する

開発者への影響

動的インコンテキスト学習の設計

顧客固有のコンテキスト (SKU マッピング、単位の好みなど) を動的に学習するインフラにより、汎用モデルを業界特化型ソリューションに変換可能

マルチモーダル入力の統合

OpenAI のテキスト、ビジョン、音声機能を単一エコシステムで活用し、多様な入力チャネルに対応する注文処理を実現

Realtime API の活用

サブ秒レイテンシの Realtime API により、電話注文のような即応性が求められるユースケースにも AI エージェントを適用可能
