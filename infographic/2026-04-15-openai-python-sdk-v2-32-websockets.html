OpenAI Python SDK v2.32.0: WebSocket 再接続とイベントハンドラの大幅強化

WebSocket クライアントにイベントハンドラの実装、未接続状態でのメッセージエンキュー、自動再接続 (reconnection) サポートの 3 つの重要な機能を同時追加。Realtime API の開発体験と堅牢性が飛躍的に向上

📋 概要 (Key Points)

🔌

WebSocket イベントハンドラの実装

WebSocket クライアントにイベントハンドラパターンが公式サポートされ、Realtime API のオーディオストリーミングやテキスト応答の処理を宣言的かつ構造化された方法で実装可能に。従来のポーリングや低レベルのコールバック管理が不要になります。

📨

未接続状態でのメッセージエンキュー

WebSocket が接続状態でない場合でもメッセージをキューに追加可能に。接続確立前やネットワーク切断中でもメッセージは内部キューに蓄積され、接続が確立または再確立された時点で自動送信されます。

🔄

WebSocket 自動再接続サポート

ネットワーク障害やサーバータイムアウト時に SDK が自動リトライ。手動の再接続ロジックが不要になり、キューに蓄積されたメッセージは再接続後に順次送信されるためセッションの継続性が保たれます。

📁

InputFileContent への detail パラメータ追加

InputFileContent に detail パラメータが追加され、画像ファイルの Vision 処理時に high / low の解像度レベルを直接指定可能に。コストとレスポンス速度のバランス調整が容易になります。

🔐

OAuthErrorCode 型の追加

OAuth 認証フローにおけるエラーコードを表す OAuthErrorCode 型が新規追加。認証エラーを型安全に処理でき、ハンドリングコードの品質と保守性が向上します。

🔄 Before / After

Before (v2.31.0 以前)

  • 手動でポーリングや低レベルコールバックを管理
  • 未接続時のメッセージ送信で例外が発生
  • ネットワーク切断時に手動で再接続ロジックを実装
  • 接続状態を常にアプリケーション側で管理
  • ファイルデータが重複パラメータとして送信されるバグ

After (v2.32.0)

  • @ws.on() で宣言的にイベントハンドラを登録
  • 内部キューにメッセージを蓄積し接続後に自動送信
  • reconnect=True で自動再接続が有効に
  • SDK が接続状態を管理しセッション継続性を維持
  • ファイルデータ送信バグを修正し安定動作

✨ 主な機能

🎯

WebSocket イベントハンドラ

@ws.on("event_type") デコレータで各イベントタイプに対する処理を個別に定義。オーディオストリーミング、テキスト応答、エラーなどのイベントを構造化された方法で処理でき、コードの可読性と保守性が大幅に向上します。

📩

メッセージエンキュー

WebSocket 接続が確立される前やネットワーク切断中でもメッセージをキューに蓄積。接続が確立または再確立された時点で自動送信されるため、タイミングに依存しない安定したメッセージ送信が可能になります。

🔄

自動再接続 (Reconnection)

ネットワークの一時的な障害やサーバー側のタイムアウトが発生した場合に SDK が自動的にリトライ。再接続時にはキューに蓄積されたメッセージが順次送信され、セッションの継続性が保たれます。不安定なネットワーク環境やモバイルアプリケーションで特に有効です。

🖼

InputFileContent detail パラメータ

画像ファイルを Vision 機能で処理する際に high や low といった解像度レベルを InputFileContent 経由で直接指定可能に。API 呼び出しの柔軟性が向上し、コストとレスポンス速度のバランスを調整できます。

🔐

OAuthErrorCode 型

OAuth 認証フローで発生しうるエラーコードを表す新しい型。認証エラーを網羅的かつ型安全に処理でき、ハンドリングコードの品質と保守性が向上します。

🐛

ファイルデータ送信バグ修正

ファイルデータ送信時にパラメータが重複して送信される問題が修正されました。大容量ファイルの送信が安定し、一部のワークアラウンドコードが不要になります。

💻 コードサンプル

Python WebSocket イベントハンドラの使用例 
from openai import OpenAI

client = OpenAI()

# Realtime API with event handler (new in v2.32.0)
async with client.realtime.connect(model="gpt-4o-realtime") as ws:
    # Register event handlers for different event types
    @ws.on("response.audio.delta")
    async def handle_audio(event):
        # Process audio stream chunks
        audio_data = event.delta
        await play_audio(audio_data)

    @ws.on("response.text.delta")
    async def handle_text(event):
        # Process text stream chunks
        print(event.delta, end="", flush=True)

    @ws.on("error")
    async def handle_error(event):
        print(f"Error occurred: {event.error}")

    # Send a message to start the conversation
    await ws.send({
        "type": "conversation.item.create",
        "item": {
            "type": "message",
            "role": "user",
            "content": [{"type": "input_text", "text": "Hello!"}]
        }
    })

    await ws.send({"type": "response.create"})
Python 未接続状態でのエンキューと再接続 
from openai import OpenAI

client = OpenAI()

# WebSocket with reconnection support (new in v2.32.0)
async with client.realtime.connect(
    model="gpt-4o-realtime",
    # Enable automatic reconnection
    reconnect=True,
) as ws:
    # Messages can be enqueued even before connection is ready
    # They will be sent automatically once connected
    await ws.send({
        "type": "session.update",
        "session": {
            "modalities": ["text", "audio"],
            "instructions": "You are a helpful assistant.",
        }
    })

    @ws.on("session.created")
    async def on_session_created(event):
        print("Session created, sending message...")
        await ws.send({
            "type": "conversation.item.create",
            "item": {
                "type": "message",
                "role": "user",
                "content": [{"type": "input_text", "text": "Tell me a joke."}]
            }
        })
        await ws.send({"type": "response.create"})

    @ws.on("response.text.done")
    async def on_text_done(event):
        print(f"\nComplete response: {event.text}")
Python InputFileContent に detail パラメータを指定 
from openai import OpenAI

client = OpenAI()

# Using detail parameter with InputFileContent (new in v2.32.0)
response = client.responses.create(
    model="gpt-4o",
    input=[
        {
            "role": "user",
            "content": [
                {
                    "type": "input_file",
                    "file_data": "data:image/png;base64,iVBORw0KGgo...",
                    "detail": "high"  # Specify detail level: "high" or "low"
                },
                {
                    "type": "input_text",
                    "text": "Describe what you see in this image in detail."
                }
            ]
        }
    ]
)

print(response.output_text)

🛠 開発者への影響

Realtime API 利用者への恩恵

  • 堅牢なリアルタイム通信: 自動再接続機能により、ネットワーク障害時にもセッションが自動的に復旧し、ユーザー体験の中断を最小限に抑えられる
  • 宣言的なイベント処理: イベントハンドラパターンにより、各イベントタイプに対する処理を個別に定義でき、コードの可読性と保守性が大幅に向上する
  • メッセージの信頼性向上: 未接続状態でもメッセージをエンキューできるため、タイミングに依存しない安定したメッセージ送信が可能になる

ファイル処理の柔軟性向上

  • Vision 処理の最適化: InputFileContent の detail パラメータにより、画像の処理解像度を明示的に制御でき、コストとレスポンス速度のバランスを調整できる
  • ファイル送信の信頼性改善: ファイルデータが重複パラメータとして送信されるバグが修正され、大容量ファイルの送信が安定する

OAuth 認証のエラーハンドリング

型安全な認証エラー処理: OAuthErrorCode 型の追加により、OAuth 認証フローにおけるエラーを網羅的かつ型安全に処理できるようになりました。

アップグレード時の注意点

  • WebSocket 関連の API は追加的な変更であり、既存コードへの破壊的影響はない
  • ファイルデータ送信のバグ修正により、一部のワークアラウンドコードが不要になる可能性がある
  • v2.31.0 からのアップグレードでは、互換性の問題は報告されていない
