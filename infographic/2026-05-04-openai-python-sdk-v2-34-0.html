API UPDATE

OpenAI Python SDK v2.34.0: Admin API Key サポートと認証機構の改善

2026-05-04

概要

  • 🔑 Admin API Key のエンドポイント単位サポート
  • 🔐 認証機構の包括的改善 (Azure, Bearer, Admin)
  • 📝 環境変数によるヘッダー設定機能の追加
  • ⚡ マルチパートリクエストのパフォーマンス最適化
  • 🔧 型の修正 (prompt_cache_retention, タイムスタンプ)

主要機能

🔑
Admin API Key per endpoint
エンドポイントごとに個別のキーを割り当て。権限分離とセキュリティ粒度の向上
🌐
環境変数ヘッダー
OPENAI_HEADERS 環境変数でカスタムヘッダーを注入。デプロイ環境ごとにコード変更不要
📂
external_key_id
プロジェクトに外部鍵 ID を設定。外部システムとの統合が容易に
🏷
email / metadata
ユーザー管理に email と metadata パラメータを追加
⚡
マルチパート最適化
ファイル構造コピーの効率化で大量ファイルアップロード高速化

コードサンプル

環境変数ヘッダー設定
import os
from openai import OpenAI

# v2.34.0: 環境変数でカスタムヘッダーを設定
os.environ["OPENAI_HEADERS"] = "X-Custom-Header=my-value"

client = OpenAI()
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello!"}]
)
Admin API Key によるプロジェクト管理
from openai import OpenAI

admin_client = OpenAI(api_key="sk-admin-...")

# v2.34.0: external_key_id でプロジェクト管理
project = admin_client.organization.projects.create(
    name="Production App",
    external_key_id="ext-key-prod-001",
)

バグ修正

修正内容 詳細
Azure 認証ヘッダー 明示的指定が可能に
prompt_cache_retention in-memory → in_memory
Bearer フォールバック Admin 認証での不適切フォールバック防止
タイムスタンプ型 created_at / completed_at を float に
ファイル配列フォーマット マルチパートの正しいフィールド名

ユースケース

1
組織管理者
きめ細かいアクセス制御、外部システム統合
2
Azure ユーザー
明示的認証ヘッダーで信頼性向上
3
大量ファイルアップロード
マルチパート最適化でパフォーマンス改善
⚠️ アップグレード時の注意点
  • prompt_cache_retention: "in-memory" → "in_memory" に変更が必要
  • Response のタイムスタンプ型変更に注意 (created_at / completed_at が float 型に)
